f:T17f7,<h2>はじめに</h2>
<p>AIエージェント（Claude Codeなど）を活用して、リモートサーバーにSSH越しでNixOSをインストールする方法を解説します。このチュートリアルでは、AIエージェントに指示を出すだけで、OSのインストールから初期設定までを自動化します。</p>
<h3>前提条件</h3>
<li>リモートサーバーにIPMIやiDRACなどのリモートコンソールアクセスがあること</li>
<li>サーバーがPXEブートまたはISOマウントに対応していること</li>
<li>ホストマシン（AIエージェントを実行するマシン）からSSH接続が可能なこと</li>
<h2>1. 準備</h2>
<h3>1.1 NixOS minimal ISOのダウンロード</h3>
<p>まず、NixOSの最小インストールISOをダウンロードします。</p>
<pre><code class="language-bash">wget https://channels.nixos.org/nixos-24.11/latest-nixos-minimal-x86_64-linux.iso
</code></pre>
<h3>1.2 リモートコンソールへのISOマウント</h3>
<p>IPMIやiDRACのWebインターフェースから、ダウンロードしたISOを仮想メディアとしてマウントします。</p>
<h3>1.3 サーバーの起動とネットワーク設定</h3>
<p>サーバーを起動し、NixOSインストーラーが起動したら、ネットワーク設定を行います。</p>
<pre><code class="language-bash">&lt;h1&gt;DHCPでIPアドレスを取得（自動の場合が多い）&lt;/h1&gt;
&lt;h1&gt;手動設定の場合&lt;/h1&gt;
ifconfig eth0 192.168.1.100 netmask 255.255.255.0 up
route add default gw 192.168.1.1
</code></pre>
<h3>1.4 SSH有効化</h3>
<p>インストーラー上でSSHを有効にします。</p>
<pre><code class="language-bash">systemctl start sshd
passwd root  # パスワードを設定（例：password）
</code></pre>
<h2>2. AIエージェントのセットアップ</h2>
<h3>2.1 ホストマシンでの準備</h3>
<p>ホストマシン（例：ローカルPC）にClaude CodeなどのAIエージェントをインストールします。</p>
<pre><code class="language-bash">&lt;h1&gt;Claude Codeの場合（例）&lt;/h1&gt;
npm install -g @anthropic-ai/claude-code
</code></pre>
<h3>2.2 SSH接続の確認</h3>
<p>ホストマシンからインストーラーにSSH接続できるか確認します。</p>
<pre><code class="language-bash">ssh root@&lt;インストーラーのIPアドレス&gt;
</code></pre>
<h2>3. AIエージェントによるNixOSインストール</h2>
<h3>3.1 エージェントへの指示</h3>
<p>AIエージェントに対して、以下のような指示を出します。</p>
<pre><code class="language-">指示: SSHでリモートサーバー（root@&lt;IP&gt;）に接続し、NixOSをインストールしてください。
&lt;li&gt;ディスクは/dev/sda（または適宜変更）&lt;/li&gt;
&lt;li&gt;パーティションはGPTで、/boot（512MB）、swap（8GB）、/（残り）&lt;/li&gt;
&lt;li&gt;ホスト名: mynixos&lt;/li&gt;
&lt;li&gt;ユーザー名: user&lt;/li&gt;
&lt;li&gt;公開鍵を設定&lt;/li&gt;
</code></pre>
<h3>3.2 AIエージェントの実行内容</h3>
<p>AIエージェントは以下のようなコマンドを自動実行します。</p>
<h4>パーティション作成</h4>
<pre><code class="language-bash">parted /dev/sda -- mklabel gpt
parted /dev/sda -- mkpart primary 512MB -8GB
parted /dev/sda -- mkpart primary linux-swap -8GB 100%
parted /dev/sda -- mkpart ESP fat32 1MB 512MB
parted /dev/sda -- set 3 esp on
</code></pre>
<h4>ファイルシステム作成</h4>
<pre><code class="language-bash">mkfs.ext4 /dev/sda1
mkswap /dev/sda2
mkfs.fat -F 32 /dev/sda3
</code></pre>
<h4>マウント</h4>
<pre><code class="language-bash">mount /dev/sda1 /mnt
mount --mkdir /dev/sda3 /mnt/boot
swapon /dev/sda2
</code></pre>
<h4>設定生成</h4>
<pre><code class="language-bash">nixos-generate-config --root /mnt
</code></pre>
<h4>configuration.nixの編集</h4>
<p>AIエージェントが自動で編集します。主な設定項目：</p>
<pre><code class="language-nix">{ config, pkgs, ... }:
<p>{
  imports = [ ./hardware-configuration.nix ];</p>
<p>boot.loader.systemd-boot.enable = true;
  boot.loader.efi.canTouchEfiVariables = true;</p>
<p>networking.hostName = "mynixos";
  networking.networkmanager.enable = true;</p>
<p>users.users.user = {
    isNormalUser = true;
    extraGroups = [ "wheel" "networkmanager" ];
    openssh.authorizedKeys.keys = [
      "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC..."
    ];
  };</p>
<p>services.openssh.enable = true;</p>
<p>system.stateVersion = "24.11";
}
</code></pre></p>
<h4>インストール実行</h4>
<pre><code class="language-bash">nixos-install
</code></pre>
<h4>再起動</h4>
<pre><code class="language-bash">reboot
</code></pre>
<h2>4. インストール後の確認</h2>
<h3>4.1 SSH再接続</h3>
<p>サーバーが再起動したら、新しいIPアドレス（DHCPの場合）に対してSSH接続します。</p>
<pre><code class="language-bash">ssh user@&lt;新しいIP&gt;
</code></pre>
<h3>4.2 AIエージェントによる初期設定</h3>
<p>AIエージェントに追加の設定を依頼できます。例：</p>
<pre><code class="language-">指示: サーバーにDockerとNginxをインストールし、ファイアウォールを設定してください。
</code></pre>
<h2>5. トラブルシューティング</h2>
<h3>SSH接続が拒否される</h3>
<li>インストーラーで<code>systemctl start sshd</code>が実行されているか確認</li>
<li>ファイアウォールがSSHポートをブロックしていないか確認</li>
<h3>インストールに失敗する</h3>
<li>ディスク名が正しいか確認（<code>lsblk</code>で確認）</li>
<li>ネットワークが安定しているか確認</li>
