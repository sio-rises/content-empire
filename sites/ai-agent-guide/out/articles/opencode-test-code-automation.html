OpenCodeとは？テストコード自動生成の基本

OpenCodeは、コード生成AIを活用して開発効率を高めるツールです。特にテストコードの自動生成に強みを持ち、TDD（テスト駆動開発）の実践を支援します。本記事では、OpenCodeを使ってJestベースのテストコードを自動生成し、品質を向上させる方法を具体的に解説します。

OpenCodeのセットアップと基本操作

インストール方法

OpenCodeはCLIツールとして提供されます。以下のコマンドでインストールします。

npm install -g opencode

インストール後、OpenCodeの設定ファイルをプロジェクトルートに作成します。

opencode init

テストコード自動生成の流れ

  • テストを書きたい関数やモジュールを指定
  • OpenCodeがコードを解析し、適切なテストケースを生成
  • 生成されたテストコードをJestで実行
  • 必要に応じて手動で修正

    • 実践：Jestテストコードの自動生成

    サンプル関数：ユーザー名のバリデーション

    以下のようなシンプルなバリデーション関数を例にします。

    // validateUsername.js
export function validateUsername(username) {
  if (typeof username !== 'string') {
    throw new Error('Username must be a string');
  }
  if (username.length < 3) {
    throw new Error('Username must be at least 3 characters');
  }
  if (username.length > 20) {
    throw new Error('Username must be at most 20 characters');
  }
  if (!/^[a-zA-Z0-9_]+$/.test(username)) {
    throw new Error('Username can only contain letters, numbers, and underscores');
  }
  return username;
}

    OpenCodeでテスト生成

    次のコマンドを実行します。

    opencode test --file validateUsername.js --framework jest

    生成されるテストコード例：

    // validateUsername.test.js
import { validateUsername } from './validateUsername';

    describe('validateUsername', () => {
  test('should return the username for valid input', () => {
    expect(validateUsername('john_doe')).toBe('john_doe');
  });
    

    test('should throw error for non-string input', () => {
    expect(() => validateUsername(123)).toThrow('Username must be a string');
    expect(() => validateUsername(null)).toThrow('Username must be a string');
    expect(() => validateUsername(undefined)).toThrow('Username must be a string');
  });
    

    test('should throw error for too short username', () => {
    expect(() => validateUsername('ab')).toThrow('Username must be at least 3 characters');
  });
    

    test('should throw error for too long username', () => {
    const longName = 'a'.repeat(21);
    expect(() => validateUsername(longName)).toThrow('Username must be at most 20 characters');
  });
    

    test('should throw error for invalid characters', () => {
    expect(() => validateUsername('user name')).toThrow('Username can only contain letters, numbers, and underscores');
    expect(() => validateUsername('user@name')).toThrow('Username can only contain letters, numbers, and underscores');
  });
});

    テスト実行

    npx jest validateUsername.test.js

    すべてのテストがパスすることを確認します。

    TDDサイクルにOpenCodeを組み込む

    TDDの「Red-Green-Refactor」サイクルにOpenCodeを活用する方法です。

  • Red: まず、実装したい機能のテストを手動で書くか、OpenCodeに「このようなテストを生成して」と指示します。
  • Green: OpenCodeのテスト生成機能を使って、テストをパスする最小限のコードを生成します。
  • Refactor: 生成されたコードをリファクタリングし、品質を高めます。

    • OpenCodeはテスト駆動開発の「Green」フェーズを高速化し、開発者が本質的なロジックに集中できるようにします。

    自動生成テストコードの品質を高めるコツ

    エッジケースの指定

    OpenCodeに生成させる際、以下のようなオプションを追加すると、より網羅的なテストが生成されます。

    opencode test --file validateUsername.js --framework jest --edge-cases

    既存のテストパターンの学習

    OpenCodeはプロジェクト内の既存テストを分析し、同じスタイルでテストを生成できます。

    opencode test --file newFunction.js --framework jest --learn-from ./tests

    生成後のレビュー

    自動生成されたテストコードは必ずレビューしましょう。以下のポイントを確認します。

  • テストケースが仕様をカバーしているか
  • モックやスタブが適切に使われているか
  • 非同期処理が正しくテストされているか

    • よくあるエラーと解決策

    エラー1: テストが実行できない

    原因: OpenCodeが生成したテストコードがJestのバージョンと互換性がない。

    解決策: package.jsonでJestのバージョンを確認し、OpenCodeの設定ファイルでバージョンを指定します。

    // opencode.config.json
{
  "jestVersion": "29"
}

    エラー2: モジュールのインポートが失敗する

    原因: OpenCodeが相対パスを正しく解決できない。

    解決策: テストファイルの生成場所を指定するか、手動でインポートパスを修正します。

    opencode test --file src/utils/validate.js --framework jest --output-dir tests/unit

    まとめ

    OpenCodeを使ったテストコードの自動生成は、TDDの実践を支援し、コード品質の向上に貢献します。ただし、自動生成に頼りすぎず、開発者自身がテストの意図を理解することが重要です。ぜひ、実際のプロジェクトでOpenCodeを試してみてください。
