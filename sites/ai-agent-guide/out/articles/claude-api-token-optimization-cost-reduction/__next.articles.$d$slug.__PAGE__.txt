2:T1c0e,<h2>はじめに</h2>
<p>Claude APIを利用する際、コストはトークン消費量に直接依存します。APIの料金は入力（プロンプト）と出力（レスポンス）のトークン数に基づいて計算されるため、トークンを効率的に使うことがコスト削減の鍵です。本記事では、Claude APIのトークン最適化とコスト削減のための実践テクニックを、具体的な方法とともに解説します。</p>
<h2>トークンとは？</h2>
<p>トークンは、テキストを分割した単位です。Claudeでは、1トークンは約3〜4文字（英語の場合）または1文字（日本語の場合）に相当します。APIリクエストの費用は、プロンプトとレスポンスの合計トークン数に応じて課金されます。</p>
<h2>プロンプトの最適化</h2>
<h3>1. プロンプトを簡潔にする</h3>
<p>不要な説明や冗長な表現を削減しましょう。例えば、以下のように改善できます。</p>
<p><strong>非効率なプロンプト:</strong>
<pre><code class="language-">あなたは優秀な翻訳者です。以下の英文を日本語に翻訳してください。翻訳の際には、自然な日本語になるように注意してください。
英文: "Hello, how are you?"
</code></pre></p>
<p><strong>最適化したプロンプト:</strong>
<pre><code class="language-">翻訳: "Hello, how are you?" → 日本語
</code></pre></p>
<p>これにより、トークン数を約30%削減できます。</p>
<h3>2. システムプロンプトを活用する</h3>
<p>Claude APIでは、システムプロンプト（<code>system</code>パラメータ）を使用して、会話の冒頭で指示を与えることができます。システムプロンプトはユーザーとアシスタントのやり取りに含まれないため、毎回同じ指示を繰り返す必要がなく、トークンを節約できます。</p>
<p><strong>例:</strong>
<pre><code class="language-json">{
  "system": "あなたは親切なアシスタントです。簡潔に回答してください。",
  "messages": [
    {"role": "user", "content": "Claude APIの使い方を教えて"}
  ]
}
</code></pre></p>
<h3>3. 指示を具体化する</h3>
<p>あいまいな指示は、Claudeが余計な推論をしてトークンを消費する原因になります。回答の形式や長さを指定しましょう。</p>
<p><strong>例:</strong>
<pre><code class="language-">箇条書きで3つのポイントを挙げてください。各ポイントは50文字以内で。
</code></pre></p>
<h2>レスポンスの制御</h2>
<h3>1. <code>max_tokens</code>を設定する</h3>
<p><code>max_tokens</code>パラメータでレスポンスの最大トークン数を制限します。過剰に長い回答を防ぎ、コストを予測可能にします。</p>
<p><strong>例:</strong>
<pre><code class="language-json">{
  "max_tokens": 100,
  "messages": [
    {"role": "user", "content": "量子コンピュータの仕組みを簡潔に説明してください"}
  ]
}
</code></pre></p>
<h3>2. <code>stop</code>シーケンスを利用する</h3>
<p>特定の文字列で生成を停止させることで、不要な部分をカットできます。例えば、JSON出力を期待する場合、<code>stop</code>に<code>"\n"</code>を設定すると、最初の改行で停止します。</p>
<p><strong>例:</strong>
<pre><code class="language-json">{
  "stop": ["\n"],
  "messages": [
    {"role": "user", "content": "次の質問に一言で答えてください: 日本の首都は？"}
  ]
}
</code></pre></p>
<h3>3. 温度パラメータを調整する</h3>
<p><code>temperature</code>を低く（0〜0.3）設定すると、より決定的で簡潔な回答になり、トークン消費が安定します。逆に高いと多様な表現になり、長くなる可能性があります。</p>
<h2>トークン管理の実践</h2>
<h3>1. トークン使用量をモニタリングする</h3>
<p>APIレスポンスの<code>usage</code>フィールドには、<code>input_tokens</code>と<code>output_tokens</code>が含まれます。これをログに記録し、分析することで、どのリクエストがコストを多く消費しているか把握できます。</p>
<p><strong>例（Python）:</strong>
<pre><code class="language-python">import anthropic</p>
<p>client = anthropic.Anthropic()
response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=100,
    messages=[{"role": "user", "content": "こんにちは"}]
)
print(response.usage)
&lt;h1&gt;{'input_tokens': 10, 'output_tokens': 20}&lt;/h1&gt;
</code></pre></p>
<h3>2. 会話履歴をトリミングする</h3>
<p>マルチターンの会話では、過去のメッセージをすべて送信するとトークンが肥大化します。古いやり取りを要約するか、重要な部分だけを残すようにしましょう。</p>
<p><strong>例:</strong>
<pre><code class="language-python">&lt;h1&gt;最新の2往復だけを保持&lt;/h1&gt;
messages = conversation_history[-4:]  # 最新4メッセージ
</code></pre></p>
<h3>3. バッチ処理を検討する</h3>
<p>複数のリクエストを1つにまとめることで、プロンプトの共通部分を削減できます。例えば、複数の質問を1回のAPIコールで送信し、回答を分割して受け取ります。</p>
<p><strong>非効率:</strong>
<pre><code class="language-">Q1: 東京の人口は？
A1: ...
Q2: 大阪の人口は？
A2: ...
</code></pre></p>
<p><strong>効率的:</strong>
<pre><code class="language-">以下の質問に順番に答えてください。
&lt;li&gt;東京の人口は？&lt;/li&gt;
&lt;li&gt;大阪の人口は？&lt;/li&gt;
</code></pre></p>
<h2>モデル選択の最適化</h2>
<p>Claudeには複数のモデルがあり、タスクに応じて適切なモデルを選ぶことでコストを削減できます。</p>
<li><strong>Claude 3 Haiku</strong>: 最も高速で低コスト。簡単なタスクや大量のリクエストに最適。</li>
<li><strong>Claude 3 Sonnet</strong>: バランスの取れた性能。一般的な用途に。</li>
<li><strong>Claude 3 Opus</strong>: 高精度だが高コスト。複雑な推論が必要な場合のみ使用。</li>
<p><strong>例:</strong>
<pre><code class="language-json">{
  "model": "claude-3-haiku-20240307",
  "max_tokens": 50,
  "messages": [{"role": "user", "content": "2+2は？"}]
}
</code></pre></p>
