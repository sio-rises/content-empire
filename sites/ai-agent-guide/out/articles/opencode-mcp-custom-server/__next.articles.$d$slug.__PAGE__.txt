2:T26eb,<h2>はじめに</h2>
<p>OpenCodeは、AIアシスタントと連携してコード生成や編集を効率化するツールですが、標準機能だけでは足りない場面もあります。そこで役立つのが<strong>MCP（Model Context Protocol）サーバー</strong>の自作です。MCPサーバーを自作することで、OpenCodeにカスタムツールや外部サービスとの連携機能を追加できます。本記事では、TypeScriptを使用してMCPサーバーを一から作成し、OpenCodeにカスタムツールを追加する手順を詳しく解説します。</p>
<h2>MCPサーバーとは</h2>
<p>MCP（Model Context Protocol）は、AIモデル（OpenCodeのAIアシスタント）と外部ツールやデータソースを接続するためのプロトコルです。MCPサーバーを自作することで、以下のようなことが可能になります。</p>
<li>独自のAPI呼び出し</li>
<li>ファイルシステム操作</li>
<li>データベースクエリ</li>
<li>外部サービスの統合</li>
<p>OpenCodeは標準でいくつかのMCPサーバーをサポートしていますが、カスタムツールを追加するには自分でサーバーを実装する必要があります。</p>
<h2>前提条件</h2>
<li>Node.js 18以上がインストール済み</li>
<li>npmまたはyarnが使用可能</li>
<li>OpenCodeがインストール済み（CLIツール）</li>
<li>TypeScriptの基礎知識</li>
<h2>ステップ1：プロジェクトのセットアップ</h2>
<p>まずは新しいディレクトリを作成し、TypeScriptプロジェクトを初期化します。</p>
<pre><code class="language-bash">mkdir my-mcp-server
cd my-mcp-server
npm init -y
npm install typescript @types/node ts-node --save-dev
npm install @modelcontextprotocol/sdk
</code></pre>
<p><code>@modelcontextprotocol/sdk</code>はMCPサーバーを構築するための公式SDKです。次にTypeScriptの設定ファイルを作成します。</p>
<pre><code class="language-bash">npx tsc --init
</code></pre>
<p><code>tsconfig.json</code>を開き、<code>outDir</code>を<code>./dist</code>、<code>rootDir</code>を<code>./src</code>に設定します。</p>
<h2>ステップ2：基本的なMCPサーバーの実装</h2>
<p><code>src/index.ts</code>を作成し、以下のコードを記述します。これは最小限のMCPサーバーで、"hello"というツールを提供します。</p>
<pre><code class="language-typescript">import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";
<p>// サーバーインスタンスを作成
const server = new Server(
  {
    name: "my-mcp-server",
    version: "1.0.0",
  },
  {
    capabilities: {
      tools: {},
    },
  }
);</p>
<p>// ツール一覧を返すハンドラー
server.setRequestHandler(ListToolsRequestSchema, async () =&gt; {
  return {
    tools: [
      {
        name: "hello",
        description: "挨拶を返すツール",
        inputSchema: {
          type: "object",
          properties: {
            name: {
              type: "string",
              description: "名前",
            },
          },
          required: ["name"],
        },
      },
    ],
  };
});</p>
<p>// ツール実行ハンドラー
server.setRequestHandler(CallToolRequestSchema, async (request) =&gt; {
  if (request.params.name === "hello") {
    const name = request.params.arguments?.name as string;
    return {
      content: [
        {
          type: "text",
          text: <code>こんにちは、${name}さん！</code>,
        },
      ],
    };
  }
  throw new Error(<code>Unknown tool: ${request.params.name}</code>);
});</p>
<p>// 標準入出力トランスポートでサーバーを起動
const transport = new StdioServerTransport();
await server.connect(transport);
</code></pre></p>
<h2>ステップ3：カスタムツールの追加</h2>
<p>次に、実際に役立つカスタムツールを追加しましょう。例として、ファイルの内容を読み取るツール<code>read_file</code>を実装します。</p>
<p><code>src/index.ts</code>を編集し、<code>ListToolsRequestSchema</code>のハンドラー内の<code>tools</code>配列に新しいツール定義を追加します。</p>
<pre><code class="language-typescript">tools: [
  {
    name: "hello",
    description: "挨拶を返すツール",
    inputSchema: {
      type: "object",
      properties: {
        name: {
          type: "string",
          description: "名前",
        },
      },
      required: ["name"],
    },
  },
  {
    name: "read_file",
    description: "指定されたファイルの内容を読み取ります",
    inputSchema: {
      type: "object",
      properties: {
        path: {
          type: "string",
          description: "ファイルのパス",
        },
      },
      required: ["path"],
    },
  },
],
</code></pre>
<p>次に、<code>CallToolRequestSchema</code>のハンドラーに<code>read_file</code>の処理を追加します。</p>
<pre><code class="language-typescript">if (request.params.name === "hello") {
  // 既存の処理
} else if (request.params.name === "read_file") {
  const path = request.params.arguments?.path as string;
  try {
    const fs = await import("fs/promises");
    const content = await fs.readFile(path, "utf-8");
    return {
      content: [
        {
          type: "text",
          text: content,
        },
      ],
    };
  } catch (error: any) {
    return {
      content: [
        {
          type: "text",
          text: <code>エラー: ${error.message}</code>,
        },
      ],
      isError: true,
    };
  }
}
</code></pre>
<p>同様に、他にも便利なツールを追加できます。例えば、現在の日時を返す<code>get_current_time</code>や、簡単な計算を行う<code>calculate</code>などが考えられます。</p>
<h2>ステップ4：ビルドと実行</h2>
<p>TypeScriptをコンパイルします。</p>
<pre><code class="language-bash">npx tsc
</code></pre>
<p>これで<code>dist/index.js</code>が生成されます。サーバーをテストするには、以下のコマンドで直接実行できます。</p>
<pre><code class="language-bash">node dist/index.js
</code></pre>
<p>ただし、MCPサーバーは標準入出力を介して通信するため、通常はOpenCodeから呼び出します。</p>
<h2>ステップ5：OpenCodeへの接続</h2>
<p>OpenCodeの設定ファイル（通常は<code>~/.opencode/config.json</code>）を編集し、MCPサーバーを追加します。</p>
<pre><code class="language-json">{
  "mcpServers": {
    "my-mcp-server": {
      "command": "node",
      "args": ["/absolute/path/to/my-mcp-server/dist/index.js"],
      "env": {}
    }
  }
}
</code></pre>
<p>パスは絶対パスで指定してください。設定後、OpenCodeを再起動すると、AIアシスタントがカスタムツールを認識し、使用できるようになります。</p>
<h2>高度なカスタマイズ</h2>
<h3>ツールの入力スキーマを詳細に設定する</h3>
<p>JSON Schemaに従って、より複雑な入力スキーマを定義できます。例えば、<code>read_file</code>にエンコーディングオプションを追加する場合：</p>
<pre><code class="language-typescript">inputSchema: {
  type: "object",
  properties: {
    path: { type: "string" },
    encoding: { type: "string", enum: ["utf-8", "ascii", "base64"] },
  },
  required: ["path"],
}
</code></pre>
<h3>リソースの提供</h3>
<p>ツールだけでなく、リソースを提供することもできます。リソースは読み取り専用のデータで、AIが参照できます。</p>
<pre><code class="language-typescript">import { ListResourcesRequestSchema, ReadResourceRequestSchema } from "@modelcontextprotocol/sdk/types.js";
<p>server.setRequestHandler(ListResourcesRequestSchema, async () =&gt; ({
  resources: [
    {
      uri: "file:///config.json",
      name: "設定ファイル",
      mimeType: "application/json",
    },
  ],
}));</p>
<p>server.setRequestHandler(ReadResourceRequestSchema, async (request) =&gt; {
  if (request.params.uri === "file:///config.json") {
    const content = JSON.stringify({ key: "value" });
    return {
      contents: [
        {
          uri: request.params.uri,
          mimeType: "application/json",
          text: content,
        },
      ],
    };
  }
  throw new Error("Resource not found");
});
</code></pre></p>
<h3>エラーハンドリングの改善</h3>
<p>ツール実行時にエラーが発生した場合、<code>isError: true</code>を設定してエラーメッセージを返すことで、AIが適切に処理できます。</p>
<h2>トラブルシューティング</h2>
<h3>サーバーが認識されない</h3>
<li>OpenCodeの設定ファイルのパスが正しいか確認</li>
<li>サーバーが実行可能かどうか、直接起動してテスト</li>
