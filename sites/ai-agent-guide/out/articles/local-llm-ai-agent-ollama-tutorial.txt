f:T1c74,<h2>はじめに</h2>
<p>近年、AIエージェントが注目を集めていますが、クラウド上のLLMを利用するとプライバシーやコストの懸念があります。そこで、ローカルで動作するLLM（ローカルLLM）とAIエージェントを組み合わせることで、オフラインかつプライベートな環境でエージェントを実行できます。本チュートリアルでは、<strong>Ollama</strong>を使ってローカルLLMをセットアップし、AIエージェントと連携する方法を具体的に解説します。</p>
<h2>ローカルLLMとAIエージェントのメリット</h2>
<li><strong>プライバシー</strong>: データが外部に送信されないため、機密情報を扱う業務に最適です。</li>
<li><strong>オフライン動作</strong>: インターネット接続が不要なため、安定した動作が可能です。</li>
<li><strong>コスト削減</strong>: API費用がかからず、一度セットアップすれば無料で使い放題です。</li>
<li><strong>カスタマイズ性</strong>: モデルやプロンプトを自由に変更でき、用途に合わせたエージェントを構築できます。</li>
<h2>必要なもの</h2>
<li>Ollamaが動作する環境（Windows, macOS, Linux）</li>
<li>Python 3.8以上</li>
<li>基本的なコマンドライン操作の知識</li>
<h2>ステップ1: Ollamaのインストールとモデル準備</h2>
<h3>1.1 Ollamaのインストール</h3>
<p><a href="https://ollama.com/" target="_blank" rel="noopener">Ollama公式サイト</a>からOSに合ったインストーラをダウンロードし、インストールします。</p>
<pre><code class="language-bash">&lt;h1&gt;Linux/macOSの場合（curl経由）&lt;/h1&gt;
curl -fsSL https://ollama.com/install.sh | sh
</code></pre>
<h3>1.2 モデルのダウンロード</h3>
<p>Ollamaは多くのモデルをサポートしています。ここでは軽量で性能の良い「Mistral」を使用します。</p>
<pre><code class="language-bash">ollama pull mistral
</code></pre>
<p>ダウンロードが完了したら、以下のコマンドで動作確認します。</p>
<pre><code class="language-bash">ollama run mistral "Hello, how are you?"
</code></pre>
<h2>ステップ2: AIエージェントの設計</h2>
<p>シンプルなタスク実行エージェントを作成します。エージェントはユーザーの指示を解析し、適切なアクションを実行します。今回は「ファイル操作」を例にします。</p>
<h3>2.1 機能一覧</h3>
<li>ファイル一覧の表示</li>
<li>ファイルの内容表示</li>
<li>ファイルの削除（確認付き）</li>
<h2>ステップ3: Pythonでエージェントを実装</h2>
<h3>3.1 必要なライブラリのインストール</h3>
<pre><code class="language-bash">pip install ollama
</code></pre>
<h3>3.2 エージェントコードの作成</h3>
<p><code>agent.py</code>を作成し、以下のコードを記述します。</p>
<pre><code class="language-python">import ollama
import os
import json
<p>&lt;h1&gt;Ollamaクライアントの初期化&lt;/h1&gt;
client = ollama.Client()</p>
<p>def get_llm_response(prompt):
    response = client.generate(model='mistral', prompt=prompt)
    return response['response']</p>
<p>def parse_command(user_input):
    # LLMに命令を解析させる
    system_prompt = """
    あなたはファイル操作アシスタントです。ユーザーの入力を解析し、以下のJSON形式で出力してください。
    {
        "action": "list | read | delete",
        "params": {
            "path": "ファイルパス"
        }
    }
    例:
    - ユーザー: "ファイル一覧を表示" → {"action": "list", "params": {"path": "."}}
    - ユーザー: "test.txtの内容を表示" → {"action": "read", "params": {"path": "test.txt"}}
    - ユーザー: "test.txtを削除" → {"action": "delete", "params": {"path": "test.txt"}}
    """
    full_prompt = f"{system_prompt}\n\nユーザー: {user_input}"
    response = get_llm_response(full_prompt)
    try:
        # JSON部分を抽出
        json_start = response.index('{')
        json_end = response.rindex('}') + 1
        json_str = response[json_start:json_end]
        return json.loads(json_str)
    except:
        return None</p>
<p>def execute_action(command):
    if command is None:
        return "コマンドを解析できませんでした。"
    action = command['action']
    path = command['params']['path']
    if action == 'list':
        files = os.listdir(path)
        return "\n".join(files)
    elif action == 'read':
        try:
            with open(path, 'r') as f:
                return f.read()
        except FileNotFoundError:
            return "ファイルが見つかりません。"
    elif action == 'delete':
        confirm = input(f"本当に{path}を削除しますか？ (y/n): ")
        if confirm.lower() == 'y':
            os.remove(path)
            return "削除しました。"
        else:
            return "削除をキャンセルしました。"
    else:
        return "不明なアクションです。"</p>
<p>if __name__ == "__main__":
    print("ファイル操作エージェントを開始します。終了するには 'exit' と入力。")
    while True:
        user_input = input("\n指示を入力: ")
        if user_input.lower() == 'exit':
            break
        command = parse_command(user_input)
        result = execute_action(command)
        print(result)
</code></pre></p>

README.md

<p>指示を入力: agent.pyの内容を表示
import ollama

</code></pre></p>
<h2>ステップ4: 応用とカスタマイズ</h2>
<h3>4.1 他のモデルへの変更</h3>
<p>Ollamaで利用可能なモデルに変更できます。例えば、日本語性能の高い「ELYZA-jp」を使用する場合：</p>
<pre><code class="language-bash">ollama pull elyza:jp8b
</code></pre>
<p>コード内の<code>model='mistral'</code>を<code>model='elyza:jp8b'</code>に変更します。</p>
<h3>4.2 エージェントの機能拡張</h3>
<li>ウェブ検索エージェント：ローカルで完結しない場合は、APIと組み合わせることも可能です。</li>
<li>スケジュール管理：カレンダー操作など。</li>
<h2>注意点</h2>
<li>ローカルLLMはクラウド版に比べて応答速度が遅い場合があります。</li>
<li>モデルのサイズによっては大量のメモリを消費します（8Bモデルで約4GB、13Bモデルで約8GB）。</li>
<li>セキュリティ：エージェントに削除権限を与える場合は注意が必要です。</li>
