2:T162a,<h2>OpenCodeのスキルシステムとは</h2>
<p>OpenCodeは、AIアシスタントとしてコード生成や解析を得意としますが、その機能は「スキル」と呼ばれるモジュールによって拡張できます。スキルシステムを活用することで、特定のタスクに特化した動作や、外部サービスとの連携が可能になります。本記事では、スキルの基本概念からカスタマイズ、拡張方法までを実践的に解説します。</p>
<h3>スキルの基本構造</h3>
<p>スキルは以下の要素で構成されます：
<li><strong>トリガー</strong>：特定のキーワードやパターンで発動</li>
<li><strong>アクション</strong>：実行される処理（コード生成、APIコールなど）</li>
<li><strong>コンテキスト</strong>：スキルが利用する変数や設定</li></p>
<p>OpenCodeは標準で複数のスキルを搭載していますが、ユーザーが独自に定義することも可能です。</p>
<h2>スキルのカスタマイズ方法</h2>
<h3>1. 設定ファイルの編集</h3>
<p>OpenCodeのスキルは、JSONまたはYAML形式の設定ファイルで管理されます。例えば、<code>skills/my-skill.yaml</code> を作成します：</p>
<pre><code class="language-yaml">name: "custom-skill"
description: "特定のフレームワーク用のコード生成"
triggers:
  - "generate:react-component"
actions:
  - type: "code-generation"
    language: "javascript"
    template: |
      import React from 'react';
      const {{componentName}} = () =&gt; {
        return &lt;div&gt;{{content}}&lt;/div&gt;;
      };
      export default {{componentName}};
context:
  componentName: "string"
  content: "string"
</code></pre>
<p>このスキルは、「generate:react-component」というトリガーで発動し、Reactコンポーネントのテンプレートを生成します。</p>
<h3>2. スキルの有効化</h3>
<p>作成したスキルを有効にするには、OpenCodeの設定でスキルディレクトリを指定します。</p>
<pre><code class="language-bash">opencode --skills-dir ./skills
</code></pre>
<p>または、GUI設定画面からスキルをインポートすることもできます。</p>
<h2>MCPによる拡張</h2>
<p>MCP（Model Context Protocol）は、OpenCodeが外部ツールやデータソースと連携するためのプロトコルです。スキルからMCPサーバーを呼び出すことで、データベース検索やファイル操作などを実行できます。</p>
<h3>MCPスキルの例</h3>
<pre><code class="language-yaml">name: "db-query"
description: "データベースにクエリを実行"
triggers:
  - "query:database"
actions:
  - type: "mcp-call"
    server: "my-db-server"
    method: "query"
    params:
      sql: "{{query}}"
context:
  query: "string"
</code></pre>
<p>このスキルは、MCPサーバー「my-db-server」に対してクエリを送信し、結果を返します。MCPサーバーは別途設定が必要ですが、一度構築すれば複数のスキルから再利用可能です。</p>
<h2>プラグインによる拡張</h2>
<p>プラグインは、スキルよりも高度な拡張方法です。プラグインはJavaScriptやPythonで記述でき、OpenCodeの内部APIにアクセスできます。</p>
<h3>プラグインの作成手順</h3>
<li><strong>プラグインディレクトリを作成</strong>：<code>plugins/my-plugin/</code></li>
<li><strong>プラグインファイルを作成</strong>：<code>main.js</code></li>
<pre><code class="language-javascript">module.exports = {
  name: 'my-plugin',
  hooks: {
    'before-response': (context) =&gt; {
      // 応答前に処理を追加
      context.response = context.response.toUpperCase();
    },
  },
};
</code></pre>
<li><strong>プラグインを登録</strong>：設定ファイルにプラグインのパスを追加します。</li>
<pre><code class="language-json">{
  "plugins": ["./plugins/my-plugin"]
}
</code></pre>
<p>プラグインは、スキルの実行前後や、メッセージの送受信時にフックできます。これにより、ログ記録や入力変換などが可能です。</p>
<h2>実践的なスキル開発チュートリアル</h2>
<h3>ステップ1：スキルの要件定義</h3>
<p>例えば、「コードレビュー支援スキル」を作成します。このスキルは、コードのスタイル違反を検出し、修正案を提示します。</p>
<h3>ステップ2：スキルファイルの作成</h3>
<pre><code class="language-yaml">name: "code-review"
description: "コードレビューを実行し、スタイル違反を指摘"
triggers:
  - "review:code"
actions:
  - type: "code-analysis"
    rules:
      - "no-unused-vars"
      - "prefer-const"
    format: "markdown"
</code></pre>
<h3>ステップ3：テストと調整</h3>
<p>実際にトリガーを実行し、期待通り動作するか確認します。必要に応じてルールを追加・変更します。</p>
<h3>ステップ4：共有・配布</h3>
<p>作成したスキルは、YAMLファイル単体で共有できます。コミュニティで公開することで、他のユーザーも利用可能です。</p>
