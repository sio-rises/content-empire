f:T18fc,<h2>OpenCodeとは</h2>
<p>OpenCodeは、ターミナル上で動作するAIエージェント型のCLIツールです。開発者が自然言語で指示を出すと、AIがコードの生成・編集・実行・デバッグを自動で行います。従来のコーディング支援ツールとは異なり、ファイル操作やシェルコマンドの実行もエージェントが自律的に行うため、開発フロー全体を効率化できます。</p>
<h3>主な特徴</h3>
<li><strong>自然言語での指示</strong>: 「ログイン機能を追加して」「テストを書いて」などの指示でコードを自動生成</li>
<li><strong>ファイル編集</strong>: 既存のコードベースを読み込み、修正や追加を自動適用</li>
<li><strong>コマンド実行</strong>: ビルド、テスト、デプロイなどをエージェントが実行</li>
<li><strong>マルチファイル対応</strong>: 複数ファイルにまたがる変更も一貫して処理</li>
<li><strong>ローカル実行</strong>: クラウドに依存せず、手元の環境で安全に動作</li>
<h2>OpenCodeのインストール方法</h2>
<p>OpenCodeはNode.js製のツールで、npmまたはHomebrewでインストールできます。</p>
<h3>前提条件</h3>
<li>Node.js 18以上</li>
<li>OpenAI APIキー（または互換性のあるLLMプロバイダー）</li>
<h3>npmを使用する場合</h3>
<pre><code class="language-bash">npm install -g opencode
</code></pre>
<h3>Homebrewを使用する場合</h3>
<pre><code class="language-bash">brew install opencode
</code></pre>
<p>インストール後、APIキーを設定します。
<pre><code class="language-bash">export OPENAI_API_KEY="sk-your-key"
</code></pre></p>
<p>永続的に設定するには、シェルの設定ファイル（.bashrcや.zshrc）に追記してください。</p>
<h2>基本的な使い方</h2>
<p>OpenCodeの基本的なワークフローは、ターミナルで<code>opencode</code>コマンドを実行し、指示を入力するだけです。</p>
<h3>1. 対話モードで開始</h3>
<pre><code class="language-bash">opencode
</code></pre>
プロンプトが表示されるので、やりたいことを自然言語で入力します。
<p>例:
<pre><code class="language-">&gt; 新しいReactコンポーネントを作成して。ボタンをクリックするとカウントが増えるシンプルなもの。
</code></pre></p>
<p>AIが自動的にファイルを作成し、コードを記述します。</p>
<h3>2. ワンショットコマンド</h3>
引数として直接指示を渡すこともできます。
<pre><code class="language-bash">opencode "このプロジェクトにESLintを設定して"
</code></pre>
<h3>3. ファイルを指定して修正</h3>
<pre><code class="language-bash">opencode --file src/app.js "このファイルにエラーハンドリングを追加して"
</code></pre>
<h3>4. コンテキストを与える</h3>
プロジェクト全体を理解させるには、<code>--context</code>オプションでディレクトリを指定します。
<pre><code class="language-bash">opencode --context ./src "全APIエンドポイントにバリデーションを追加して"
</code></pre>
<h2>実践的なユースケース</h2>
<h3>コード生成</h3>
「PythonでCSVを読み込んで、特定の列の平均を計算するスクリプトを作って」といった要求を自然言語で行うと、AIが即座にコードを生成し、ファイルに保存します。
<h3>リファクタリング</h3>
「この関数を非同期に書き換えて」「クラスコンポーネントを関数コンポーネントに変換して」など、コードの改善も指示一つで可能です。
<h3>バグ修正</h3>
エラーメッセージをそのまま貼り付けて「このエラーを修正して」と指示すると、AIが原因を特定し、修正コードを提案・適用します。
<h3>テスト作成</h3>
既存のコードに対して「ユニットテストを書いて」と指示するだけで、テストファイルが生成されます。
<h2>設定とカスタマイズ</h2>
<p>OpenCodeは設定ファイル（<code>opencode.config.json</code>）で動作をカスタマイズできます。</p>
<pre><code class="language-json">{
  "model": "gpt-4",
  "temperature": 0.2,
  "maxTokens": 4096,
  "systemPrompt": "あなたは優秀なソフトウェアエンジニアです。簡潔で効率的なコードを書いてください。"
}
</code></pre>
<li><code>model</code>: 使用するLLMモデル（デフォルトはgpt-4）</li>
<li><code>temperature</code>: 出力のランダム性（0〜1）</li>
<li><code>maxTokens</code>: 最大トークン数</li>
<li><code>systemPrompt</code>: AIへのシステムプロンプト（役割やルールを指定）</li>
<h2>注意点とベストプラクティス</h2>
<li><strong>APIキーの管理</strong>: キーは環境変数で管理し、公開リポジトリにコミットしない</li>
<li><strong>コードの確認</strong>: AIが生成したコードは必ず人間がレビューする（特にセキュリティ面）</li>
<li><strong>段階的な指示</strong>: 複雑なタスクは小さなステップに分割して指示する</li>
<li><strong>コンテキストの明示</strong>: プロジェクト構造やコーディング規約を事前に伝えると精度が向上</li>
<li><strong>バージョン管理</strong>: AIによる変更はGitで管理し、必要に応じてロールバックできるようにする</li>
<h2>まとめ</h2>
<p>OpenCodeは、AIエージェントを活用してコーディング作業を大幅に効率化するCLIツールです。インストールも簡単で、自然言語で指示を出すだけでコード生成・編集・実行まで自動化できます。ただし、生成されたコードの品質やセキュリティは最終的に開発者が責任を持つ必要があります。まずは小さなタスクから試してみて、徐々にワークフローに組み込んでいくことをおすすめします。</p>
