2:T1aad,<h2>はじめに</h2>
<p>サーバー管理の現場では、日々の運用タスク（ログ監視、パッケージ更新、バックアップ、トラブルシューティング）に多くの時間が費やされています。ClaudeCodeは、AIエージェントがターミナル上で直接コードを実行・編集できるツールです。本記事では、ClaudeCodeをSSH経由でリモートサーバーと連携させ、サーバー管理を自動化・効率化する具体的な方法を紹介します。</p>
<h2>前提条件</h2>
<li>ClaudeCodeがインストール済み（<code>pip install claude-code</code> など）</li>
<li>管理対象サーバーへのSSHアクセス権限</li>
<li>基本的なLinuxコマンドの知識</li>
<h2>1. ClaudeCodeからSSHでサーバーに接続する</h2>
<p>ClaudeCodeはターミナル上で動作するため、SSH接続も自然に行えます。以下のように、ClaudeCodeのセッション内で直接SSHコマンドを実行できます。</p>
<pre><code class="language-bash">&lt;h1&gt;ClaudeCode起動&lt;/h1&gt;
claude
<p>&lt;h1&gt;セッション内でSSH接続&lt;/h1&gt;
ssh user@your-server.com
</code></pre></p>
<p>ClaudeCodeは実行結果を認識し、その後の指示に反映できます。例えば、接続後に「サーバーのディスク使用率を確認して」と依頼すれば、<code>df -h</code> を実行して結果を表示します。</p>
<h2>2. よくある管理タスクの自動化</h2>
<h3>2.1 ディスク使用量の監視とアラート</h3>
<p>ClaudeCodeに以下のような指示を出すだけで、スクリプトを作成・実行してくれます。</p>
<pre><code class="language-">「/var/log以下のログファイルで、7日以上経過したものを自動的に圧縮・削除するスクリプトを作成して。cronで毎日実行できるように設定して。」
</code></pre>
<p>ClaudeCodeは以下のようなスクリプトを生成し、実行します。</p>
<pre><code class="language-bash">#!/bin/bash
&lt;h1&gt;7日以上前のログを圧縮&lt;/h1&gt;
find /var/log -name "*.log" -mtime +7 -exec gzip {} \;
&lt;h1&gt;30日以上前の圧縮済みログを削除&lt;/h1&gt;
find /var/log -name "*.gz" -mtime +30 -delete
</code></pre>
<p>その後、<code>crontab -e</code> を使って毎日午前3時に実行するよう設定します。</p>
<h3>2.2 パッケージの一括アップデート</h3>
<p>「全てのパッケージをアップデートして、アップデート後に再起動が必要かどうか教えて」</p>
<p>ClaudeCodeは <code>sudo apt update && sudo apt upgrade -y</code> を実行し、<code>/var/run/reboot-required</code> の有無を確認して結果を報告します。</p>
<h3>2.3 プロセス監視と自動再起動</h3>
<p>「nginxプロセスが落ちていたら自動で再起動するスクリプトを作成して」</p>
<p>ClaudeCodeは以下のような監視スクリプトを生成します。</p>
<pre><code class="language-bash">#!/bin/bash
if ! pgrep nginx &gt; /dev/null; then
    systemctl restart nginx
    echo "nginx再起動: $(date)" &gt;&gt; /var/log/nginx_restart.log
fi
</code></pre>
<p>これをcronで1分ごとに実行する設定まで行えます。</p>
<h2>3. 複数サーバーの一括管理</h2>
<p>ClaudeCodeの強力な点は、複数のサーバーに対して同じ操作を反復できることです。例えば、ホスト一覧をファイルに書き出しておき、以下のように指示します。</p>
<pre><code class="language-">「servers.txtに書かれた各サーバーにSSHで接続し、ディスク使用率が80%以上のものをリストアップして」
</code></pre>
<p>ClaudeCodeはループ処理で各サーバーに接続し、<code>df -h</code> の結果をパースして条件に合うものを表示します。</p>
<h2>4. ログ解析とトラブルシューティング</h2>
<p>サーバーで問題が発生した場合、ClaudeCodeにログを読ませて分析させることができます。</p>
<pre><code class="language-">「/var/log/syslogの直近100行を表示して、エラーや警告を抽出して」
</code></pre>
<p>ClaudeCodeはログを読み込み、パターンマッチングでエラー行を抜き出し、必要であれば解決策を提案します。さらに「このエラーの原因を調べて」と依頼すれば、関連する設定ファイルを確認したり、フォーラムを検索する代わりに知識ベースから回答を生成します。</p>
<h2>5. バックアップの自動化</h2>
<p>「/var/www と /etc を毎日バックアップして、7日分保持するスクリプトを作成して」</p>
<p>ClaudeCodeは以下のようなスクリプトを生成します。</p>
<pre><code class="language-bash">#!/bin/bash
BACKUP_DIR="/backup/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR
tar czf $BACKUP_DIR/www.tar.gz /var/www
tar czf $BACKUP_DIR/etc.tar.gz /etc
&lt;h1&gt;7日以上前のバックアップを削除&lt;/h1&gt;
find /backup -type d -mtime +7 -exec rm -rf {} \;
</code></pre>
<p>さらに、リモートサーバーへの転送や暗号化も指示できます。</p>
<h2>6. 注意点とベストプラクティス</h2>
<li><strong>権限管理</strong>: ClaudeCodeにroot権限を与える場合は注意が必要です。可能であればsudo権限を制限したユーザーを使い、必要なコマンドだけ許可しましょう。</li>
<li><strong>確認プロンプト</strong>: 重要な操作（削除や再起動）は、ClaudeCodeに「実行前に確認を求めて」と指示することで安全に運用できます。</li>
<li><strong>セッションの分離</strong>: 管理対象サーバーごとにClaudeCodeセッションを分けると、誤操作を防げます。</li>
<li><strong>ログの記録</strong>: ClaudeCodeの出力をファイルに保存しておくと、監査やトラブルシューティングに役立ちます。</li>
