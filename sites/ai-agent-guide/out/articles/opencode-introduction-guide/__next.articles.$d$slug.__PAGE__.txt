2:T152c,<h2>OpenCodeとは</h2>
<p>OpenCodeは、ターミナル上で動作するAIエージェントベースのコーディング支援CLIツールです。コードの生成、リファクタリング、説明、テスト作成などを、自然言語の指示で実行できます。内部では大規模言語モデル（LLM）を利用し、ファイルシステムの読み書き、コマンド実行、Git操作などをエージェントが自律的に行います。</p>
<p>従来のコード補完ツールと異なり、OpenCodeは複数ファイルにまたがるタスクや、シェルコマンドを組み合わせた複雑な作業も指示一つで完了できます。</p>
<h2>主な機能</h2>
<li><strong>コード生成</strong>：自然言語の指示からコードを生成し、ファイルに保存</li>
<li><strong>コード編集</strong>：既存のコードを読み取り、修正やリファクタリングを実行</li>
<li><strong>コード説明</strong>：選択したコードの動作を解説</li>
<li><strong>テスト作成</strong>：既存コードに対するテストコードを自動生成</li>
<li><strong>Git操作</strong>：コミットメッセージの生成、差分の説明など</li>
<li><strong>シェルコマンド実行</strong>：必要なコマンドを自動で実行（ユーザー確認あり）</li>
<h2>インストール</h2>
<p>OpenCodeはNode.js（バージョン18以上）が必要です。以下のコマンドでインストールします。</p>
<pre><code class="language-bash">npm install -g @opencode/cli
</code></pre>
<p>インストール後、OpenAIのAPIキーが必要です。環境変数に設定するか、初回実行時のプロンプトで入力します。</p>
<pre><code class="language-bash">export OPENAI_API_KEY="sk-..."
</code></pre>
<p>または、<code>.env</code>ファイルを作成して記述しても構いません。</p>
<h2>基本的な使い方</h2>
<h3>1. 対話モードで起動</h3>
<p>プロジェクトのルートディレクトリで以下のコマンドを実行します。</p>
<pre><code class="language-bash">opencode
</code></pre>
<p>対話モードが起動し、プロンプトが表示されます。ここに自然言語で指示を入力します。</p>
<h3>2. コード生成の例</h3>
<p>「Pythonでフィボナッチ数列を計算する関数を書いて」と入力すると、エージェントがコードを生成し、ファイルに保存するか提案します。</p>
<pre><code class="language-">&gt; Pythonでフィボナッチ数列を計算する関数を書いて
</code></pre>
<p>エージェントは<code>fibonacci.py</code>を作成し、中身を表示します。</p>
<h3>3. コード編集の例</h3>
<p>既存のファイルに対して、「この関数にエラーハンドリングを追加して」と指示すると、該当箇所を読み取り修正します。</p>
<pre><code class="language-">&gt; src/utils.py の fetch_data 関数にエラーハンドリングを追加して
</code></pre>
<h3>4. 複数ファイルにまたがるタスク</h3>
<p>「新しいAPIエンドポイントを追加して、それに対応するテストも書いて」といった複雑な指示も可能です。</p>
<h2>設定ファイル</h2>
<p>プロジェクトルートに<code>opencode.config.json</code>を作成することで、詳細な設定が可能です。</p>
<pre><code class="language-json">{
  "model": "gpt-4",
  "include": ["src/<strong>/*.py", "tests/</strong>/*.py"],
  "exclude": ["node_modules", "__pycache__"],
  "autoConfirm": false
}
</code></pre>
<li><code>model</code>: 使用するLLM（デフォルトはgpt-4）</li>
<li><code>include</code>: エージェントがアクセスできるファイルのパターン</li>
<li><code>exclude</code>: 除外するファイルパターン</li>
<li><code>autoConfirm</code>: 自動確認モード（危険な操作も自動実行）</li>
<h2>注意点とベストプラクティス</h2>
<li><strong>コードレビューを忘れずに</strong>：AIが生成したコードは必ず人間が確認しましょう。特にセキュリティやロジックの正確性をチェックしてください。</li>
<li><strong>機密情報を入力しない</strong>：APIキーやパスワードを会話に含めないでください。</li>
<li><strong>段階的に指示する</strong>：複雑なタスクは小さなステップに分割すると成功率が上がります。</li>
<li><strong>コンテキストを適切に与える</strong>：関連するファイルやコードの構造を事前に説明すると良い結果が得られます。</li>
