f:T154f,<h2>はじめに</h2>
<p>OpenCodeは、オープンソースのコード管理・実行プラットフォームです。このチュートリアルでは、GitHub Actionsを使ってOpenCodeプロジェクトのCI/CDパイプラインを構築する方法をステップバイステップで解説します。自動テスト、ビルド、デプロイを自動化し、開発効率を向上させましょう。</p>
<h2>前提条件</h2>
<li>GitHubアカウント</li>
<li>OpenCodeプロジェクト（リポジトリにコードがプッシュされていること）</li>
<li>基本的なGitとGitHub Actionsの知識</li>
<h2>ステップ1: GitHub Actionsのワークフローファイルを作成する</h2>
<p>リポジトリのルートに<code>.github/workflows</code>ディレクトリを作成し、その中にYAMLファイル（例：<code>ci-cd.yml</code>）を配置します。</p>
<pre><code class="language-yaml">name: CI/CD Pipeline
<p>on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]</p>
<p>jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup OpenCode
        run: |
          curl -fsSL https://opencode.sh/install.sh | bash
          opencode --version
      - name: Run tests
        run: opencode test</p>
<p>deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to production
        run: opencode deploy --env production
</code></pre></p>
<h2>ステップ2: ワークフローのトリガーを理解する</h2>
<li><code>on.push</code>：mainブランチへのプッシュ時にパイプラインを起動</li>
<li><code>on.pull_request</code>：mainブランチへのPR作成時にもテストを実行</li>
<li><code>jobs</code>：並列または直列で実行するジョブを定義</li>
<h2>ステップ3: テストジョブの設定</h2>
<p>テストジョブでは、OpenCodeをインストールし、プロジェクトのテストを実行します。<code>opencode test</code>コマンドがテストスイートを実行することを前提としています。必要に応じて、テストフレームワーク（Jest、Mochaなど）の設定を別途行ってください。</p>
<h2>ステップ4: デプロイジョブの設定</h2>
<p>デプロイジョブはテストジョブの成功後に実行され、<code>opencode deploy</code>コマンドで本番環境にデプロイします。環境変数やシークレットはGitHub Secretsに保存し、ワークフロー内で安全に参照します。</p>
<pre><code class="language-yaml">&lt;li&gt;name: Deploy&lt;/li&gt;
  env:
    OPENCODE_API_KEY: ${{ secrets.OPENCODE_API_KEY }}
  run: opencode deploy --env production
</code></pre>
<h2>ステップ5: シークレットの設定</h2>
<p>GitHubリポジトリの「Settings」→「Secrets and variables」→「Actions」で、以下のシークレットを追加します。</p>
<li><code>OPENCODE_API_KEY</code>：OpenCodeのAPIキー</li>
<li>その他必要な環境変数</li>
<h2>ステップ6: ワークフローのテスト</h2>
<p>ワークフローファイルをコミットしてプッシュすると、自動的にパイプラインが実行されます。GitHubの「Actions」タブで実行状況を確認できます。</p>
<h2>高度な設定</h2>
<h3>マルチブランチ対応</h3>
<p>複数のブランチで異なるデプロイ先を設定する場合：</p>
<pre><code class="language-yaml">&lt;li&gt;name: Deploy&lt;/li&gt;
  run: |
    if [ "${{ github.ref }}" == "refs/heads/main" ]; then
      opencode deploy --env production
    elif [ "${{ github.ref }}" == "refs/heads/staging" ]; then
      opencode deploy --env staging
    fi
</code></pre>
<h3>キャッシュの利用</h3>
<p>依存関係のインストールを高速化するためにキャッシュを追加：</p>
<pre><code class="language-yaml">&lt;li&gt;name: Cache OpenCode dependencies&lt;/li&gt;
  uses: actions/cache@v3
  with:
    path: ~/.opencode
    key: ${{ runner.os }}-opencode-${{ hashFiles('**/opencode.lock') }}
</code></pre>
<h2>トラブルシューティング</h2>
<li><strong>ワークフローが実行されない</strong>：YAMLのインデントが正しいか確認。<code>on</code>の指定が正しいか確認。</li>
<li><strong>テストが失敗する</strong>：ローカルで<code>opencode test</code>が通るか確認。環境変数の不足がないか確認。</li>
<li><strong>デプロイに失敗する</strong>：APIキーが正しく設定されているか確認。デプロイ先の設定を確認。</li>
<h2>まとめ</h2>
<p>このチュートリアルでは、OpenCodeとGitHub Actionsを連携させたCI/CDパイプラインの構築方法を紹介しました。自動テストと自動デプロイにより、開発サイクルを高速化し、ヒューマンエラーを減らすことができます。さらに高度な設定については、OpenCodeの公式ドキュメントを参照してください。</p>
