2:T203c,<h2>はじめに</h2>
<p>このチュートリアルでは、Arch Linuxにbtrfsファイルシステムを導入し、サブボリュームを作成した上で、snapperを使ってスナップショットの自動取得・管理を設定します。スナップショットを使うことで、システム更新前の状態に簡単に戻せるようになり、バックアップとしても活用できます。</p>
<h3>目標</h3>
<li>btrfsサブボリュームの作成とマウント</li>
<li>snapperのインストールと設定</li>
<li>自動スナップショットの有効化</li>
<li>スナップショットからの復元方法の理解</li>
<h3>前提条件</h3>
<li>Arch Linuxがインストール済みであること（インストール時にbtrfsを選択するか、後から変換しても構いません）</li>
<li>root権限で操作できること</li>
<li>基本的なLinuxコマンドの知識</li>
<h2>ステップ1: btrfsサブボリュームの作成</h2>
<p>まず、既存のbtrfsファイルシステム上にサブボリュームを作成します。ここでは、ルートファイルシステムが<code>/dev/sda2</code>にマウントされていると仮定します。適宜、自分の環境に合わせてデバイスを変更してください。</p>
<pre><code class="language-bash">&lt;h1&gt;マウントポイントを確認&lt;/h1&gt;
lsblk
</code></pre>
<p>ルートパーティションを一時的にマウントし、サブボリュームを作成します。</p>
<pre><code class="language-bash">&lt;h1&gt;ルートパーティションを/mntにマウント&lt;/h1&gt;
mount /dev/sda2 /mnt
<p>&lt;h1&gt;@と@homeのサブボリュームを作成（@はルート、@homeは/home用）&lt;/h1&gt;
btrfs subvolume create /mnt/@
btrfs subvolume create /mnt/@home</p>
<p>&lt;h1&gt;アンマウント&lt;/h1&gt;
umount /mnt
</code></pre></p>
<p>次に、これらのサブボリュームを適切なマウントポイントにマウントします。<code>/etc/fstab</code>を編集して、起動時に自動マウントされるようにします。</p>
<pre><code class="language-bash">&lt;h1&gt;fstabを編集&lt;/h1&gt;
nano /etc/fstab
</code></pre>
<p>以下の行を追加します（デバイス名は環境に合わせて変更）。</p>
<pre><code class="language-">/dev/sda2  /  btrfs  rw,noatime,compress=zstd,subvol=@  0  0
/dev/sda2  /home  btrfs  rw,noatime,compress=zstd,subvol=@home  0  0
</code></pre>
<p>マウントオプションの説明:
<li><code>noatime</code>: アクセス時刻を更新しない（パフォーマンス向上）</li>
<li><code>compress=zstd</code>: Zstandard圧縮を有効化</li>
<li><code>subvol=@</code>: 使用するサブボリュームを指定</li></p>
<p>マウントして確認します。</p>
<pre><code class="language-bash">&lt;h1&gt;マウント&lt;/h1&gt;
mount -a
<p>&lt;h1&gt;確認&lt;/h1&gt;
df -h
</code></pre></p>
<h2>ステップ2: snapperのインストールと設定</h2>
<p>snapperはbtrfsスナップショットを管理するツールです。</p>
<pre><code class="language-bash">&lt;h1&gt;snapperをインストール&lt;/h1&gt;
pacman -S snapper
</code></pre>
<p>snapperの設定ファイルを作成します。ルートファイルシステム（<code>/</code>）用の設定を作成します。</p>
<pre><code class="language-bash">&lt;h1&gt;設定を作成（設定名はroot）&lt;/h1&gt;
snapper -c root create-config /
</code></pre>
<p>このコマンドで、<code>/etc/snapper/configs/root</code>が作成され、スナップショットは<code>/.snapshots</code>に保存されます。</p>
<p>設定を編集して、スナップショットの保持数を調整します。</p>
<pre><code class="language-bash">nano /etc/snapper/configs/root
</code></pre>
<p>以下の項目を編集します（例）。</p>
<pre><code class="language-">SUBVOLUME="/"
ALLOW_USERS=""
ALLOW_GROUPS=""
TIMELINE_CREATE="yes"
TIMELINE_CLEANUP="yes"
TIMELINE_MIN_AGE="1800"
TIMELINE_LIMIT_HOURLY="5"
TIMELINE_LIMIT_DAILY="7"
TIMELINE_LIMIT_WEEKLY="0"
TIMELINE_LIMIT_MONTHLY="0"
TIMELINE_LIMIT_YEARLY="0"
NUMBER_LIMIT="50"
NUMBER_LIMIT_IMPORTANT="10"
</code></pre>
<li><code>TIMELINE_CREATE</code>: タイムラインスナップショットを有効にする</li>
<li><code>TIMELINE_LIMIT_*</code>: 各期間の最大保持数</li>
<li><code>NUMBER_LIMIT</code>: 手動スナップショットの最大保持数</li>
<h2>ステップ3: 自動スナップショットの有効化</h2>
<p>snapperにはタイマーが付属しており、定期的にスナップショットを作成します。</p>
<pre><code class="language-bash">&lt;h1&gt;snapper-timeline.timerを有効化&lt;/h1&gt;
systemctl enable snapper-timeline.timer
systemctl start snapper-timeline.timer
<p>&lt;h1&gt;snapper-cleanup.timerも有効化（古いスナップショットを削除）&lt;/h1&gt;
systemctl enable snapper-cleanup.timer
systemctl start snapper-cleanup.timer
</code></pre></p>
<p>タイマーの状態を確認します。</p>
<pre><code class="language-bash">systemctl list-timers | grep snapper
</code></pre>
<h2>ステップ4: スナップショットの管理</h2>
<h3>手動スナップショットの作成</h3>
<pre><code class="language-bash">&lt;h1&gt;スナップショットを作成（説明付き）&lt;/h1&gt;
snapper -c root create -d "更新前のバックアップ"
</code></pre>
<h3>スナップショットの一覧表示</h3>
<pre><code class="language-bash">snapper -c root list
</code></pre>
<p>出力例:
<pre><code class="language-"> # | Type   | Pre # | Date                            | User | Cleanup  | Description
---+--------+-------+---------------------------------+------+----------+-------------
0  | single |       |                                 | root |          | current
1  | single |       | Tue 01 Jan 2023 12:00:00 AM UTC | root | timeline | timeline
</code></pre></p>
<h3>スナップショットの削除</h3>
<pre><code class="language-bash">&lt;h1&gt;スナップショット番号1を削除&lt;/h1&gt;
snapper -c root delete 1
</code></pre>
<h2>ステップ5: スナップショットからの復元</h2>
<p>注意: 復元は慎重に行ってください。現在のデータが失われる可能性があります。</p>
<pre><code class="language-bash">&lt;h1&gt;スナップショット番号1にロールバック&lt;/h1&gt;
snapper -c root undochange 1..0
</code></pre>
<p>または、スナップショットを別のディレクトリにマウントして必要なファイルだけを取り出すこともできます。</p>
<pre><code class="language-bash">&lt;h1&gt;スナップショットをマウント&lt;/h1&gt;
mount -o subvol=.snapshots/1/snapshot /dev/sda2 /mnt/snapshot
</code></pre>
<h2>動作確認</h2>
<li>手動スナップショットを作成し、<code>snapper list</code>で表示されることを確認。</li>
<li>タイマーが動作しているか<code>journalctl -u snapper-timeline.timer</code>で確認。</li>
<li>テストファイルを作成し、スナップショットを取得後、ファイルを削除してから<code>snapper undochange</code>で復元できることを確認。</li>
