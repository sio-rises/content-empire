2:T154c,<h2>はじめに</h2>
<p>Arch Linuxを使っていると、いつの間にかディスク容量が減っていることに気づくことがあります。特にパッケージ管理のpacmanが保存するキャッシュは、アップデートを繰り返すうちに数GBにも膨れ上がる可能性があります。この記事では、ディスク容量の確認方法と、安全に掃除する手順を紹介します。</p>
<h2>ディスク容量の確認</h2>
<p>まず、現在のディスク使用状況を把握しましょう。</p>
<h3><code>df</code> コマンドで全体を確認</h3>
<pre><code class="language-bash">df -h
</code></pre>
<li><code>-h</code> は人間が読みやすい形式（GB, MB）で表示します。</li>
<li>出力例:</li>
  <pre><code class="language-">  Filesystem      Size  Used Avail Use% Mounted on
  /dev/sda1        50G   30G   20G  60% /
  </code></pre>
<li><code>/</code> のマウントポイントの使用率が高い場合、掃除が必要です。</li>
<h3><code>du</code> コマンドでディレクトリごとに確認</h3>
<pre><code class="language-bash">du -sh /var/cache/pacman/pkg
</code></pre>
<li><code>-s</code> は合計サイズのみ表示、<code>-h</code> は人間が読みやすい形式。</li>
<li>このコマンドでpacmanキャッシュのサイズがわかります。</li>
<p>他の大きなディレクトリを探すには:</p>
<pre><code class="language-bash">du -sh /var/* | sort -rh | head -10
</code></pre>
<li>パイプで <code>sort -rh</code> に渡すことで、サイズの大きい順に並べ替え。</li>
<h2>pacmanキャッシュの掃除</h2>
<h3>pacmanキャッシュとは</h3>
<p><code>/var/cache/pacman/pkg/</code> には、インストール・アップデートしたパッケージのアーカイブ（.pkg.tar.zst）が保存されています。デフォルトでは過去のバージョンも保持するため、不要なものを削除することで容量を解放できます。</p>
<h3><code>paccache</code> で安全に削除</h3>
<p><code>paccache</code> はpacmanのキャッシュ管理ツールです。インストールされていない場合:</p>
<pre><code class="language-bash">sudo pacman -S pacman-contrib
</code></pre>
<p>基本的な使い方:</p>
<pre><code class="language-bash">sudo paccache -r
</code></pre>
<li><code>-r</code> は削除（remove）オプション。デフォルトでは最新3バージョン以外のキャッシュを削除します。</li>
<p>もっと積極的に削除したい場合（最新1バージョンのみ保持）:</p>
<pre><code class="language-bash">sudo paccache -rk 1
</code></pre>
<li><code>-k</code> で保持するバージョン数を指定。</li>
<p>削除前に何が削除されるか確認するには:</p>
<pre><code class="language-bash">sudo paccache -d
</code></pre>
<li><code>-d</code> はドライラン（削除は実行しない）。</li>
<h3>未使用パッケージのキャッシュも削除</h3>
<p>インストールされていないパッケージのキャッシュを削除:</p>
<pre><code class="language-bash">sudo paccache -ruk 0
</code></pre>
<li><code>-u</code> は未インストール（uninstalled）パッケージのみ対象。</li>
<li><code>-k 0</code> でゼロバージョン保持＝全て削除。</li>
<h3>キャッシュを全削除（注意）</h3>
<pre><code class="language-bash">sudo rm -rf /var/cache/pacman/pkg/*
</code></pre>
<li>ただし、ダウングレードが必要になった場合に困るので、<code>paccache</code> を使うことを推奨します。</li>
<h2>その他の掃除場所</h2>
<h3>ジャーナルログ</h3>
<pre><code class="language-bash">sudo journalctl --vacuum-size=100M
</code></pre>
<li>ログを100MBまでに制限。</li>
<h3>不要なパッケージの削除</h3>
<pre><code class="language-bash">sudo pacman -Rns $(pacman -Qdtq)
</code></pre>
<li><code>-Qdtq</code> で孤立した依存関係を一覧表示。<code>-Rns</code> で設定ファイルも含め削除。</li>
<h2>トラブルシューティング</h2>
<h3>問題1: <code>paccache</code> が存在しない</h3>
<pre><code class="language-bash">sudo pacman -S pacman-contrib
</code></pre>
<h3>問題2: 削除後も容量が増えない</h3>
<li>削除したファイルがまだプロセスによって使用されている可能性があります。再起動後に確認。</li>
<li><code>du</code> で再確認。</li>
<h3>問題3: <code>du</code> の結果と <code>df</code> の空き容量が一致しない</h3>
<li>削除してもプロセスがファイルを掴んでいると解放されません。<code>lsof | grep deleted</code> で確認。</li>
