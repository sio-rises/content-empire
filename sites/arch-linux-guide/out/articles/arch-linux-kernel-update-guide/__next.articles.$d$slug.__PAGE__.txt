2:T1c4b,<h2>はじめに：Arch Linuxのカーネルアップデートとは</h2>
<p>Arch Linuxはローリングリリースモデルを採用しており、カーネルを含む全てのパッケージが頻繁にアップデートされます。カーネルアップデートはシステムの安定性やセキュリティに直結するため、その仕組みを理解することはArchユーザーにとって重要です。本記事では、カーネルアップデートの定義、背景、仕組み、実用例、よくある誤解を解説します。</p>
<h2>定義：カーネルアップデートと関連用語</h2>
<li><strong>カーネル（Kernel）</strong>：OSの中核で、ハードウェアとソフトウェアの橋渡しを行います。Arch Linuxではデフォルトで<code>linux</code>パッケージ（最新安定版）がインストールされます。</li>
<li><strong>linux-lts</strong>：Long Term Support（長期サポート）版のカーネル。安定性重視で、最新機能よりも長期間のサポートを提供します。</li>
<li><strong>mkinitcpio</strong>：Arch Linuxでinitramfs（初期RAMファイルシステム）を生成するツール。カーネルが起動時に必要なドライバやモジュールをロードするために使用します。</li>
<li><strong>fallback</strong>：<code>mkinitcpio</code>が生成する予備のinitramfsイメージ。デフォルトのinitramfsで起動できない場合に使用します。</li>
<h2>背景：なぜカーネルアップデートが重要なのか</h2>
<p>Arch Linuxのローリングリリースでは、カーネルは頻繁に更新されます。これにより、最新のハードウェアサポート、セキュリティパッチ、バグ修正が迅速に提供されます。しかし、アップデート後にシステムが起動しなくなるリスクもあるため、<code>linux-lts</code>のような安定版カーネルを併用するユーザーも多いです。また、<code>mkinitcpio</code>はカーネルアップデート時に自動的に実行され、新しいカーネルに対応したinitramfsを生成します。</p>
<h2>仕組み：カーネルアップデートの流れとmkinitcpioの役割</h2>
<h3>1. パッケージのアップデート</h3>
<code>pacman -Syu</code>を実行すると、<code>linux</code>パッケージが更新されます。このとき、<code>mkinitcpio</code>がフックとして自動的に起動し、新しいカーネル用のinitramfsを生成します。
<h3>2. mkinitcpioの動作</h3>
<code>mkinitcpio</code>は<code>/etc/mkinitcpio.conf</code>の設定に基づいて、カーネルモジュールやフック（例：<code>udev</code>、<code>systemd</code>、<code>keyboard</code>など）を含むinitramfsを作成します。生成されるファイルは<code>/boot/initramfs-linux.img</code>（デフォルト）と<code>/boot/initramfs-linux-fallback.img</code>（フォールバック）です。
<h3>3. fallbackイメージの役割</h3>
<code>fallback</code>イメージは、デフォルトのinitramfsに含まれない汎用的なモジュールを含む安全策です。例えば、ストレージドライバがデフォルトで不足している場合でも、fallbackイメージを使えば起動できる可能性があります。ブートローダー（GRUBなど）で起動時に選択可能です。
<h3>4. linux-ltsカーネルの併用</h3>
<code>linux-lts</code>パッケージをインストールすると、<code>/boot/vmlinuz-linux-lts</code>と<code>/boot/initramfs-linux-lts.img</code>が生成されます。ブートローダーでデフォルトカーネルとLTSカーネルを切り替えられます。
<h2>実用例：カーネルアップデートの実際の手順</h2>
<h3>例1：通常のカーネルアップデート</h3>
<pre><code class="language-bash">&lt;h1&gt;システム全体をアップデート&lt;/h1&gt;
sudo pacman -Syu
</code></pre>
これにより、<code>linux</code>パッケージが更新され、<code>mkinitcpio</code>が自動的に新しいinitramfsを生成します。再起動後、新しいカーネルで起動します。
<h3>例2：linux-ltsカーネルのインストールと設定</h3>
<pre><code class="language-bash">&lt;h1&gt;linux-ltsパッケージをインストール&lt;/h1&gt;
sudo pacman -S linux-lts
<p>&lt;h1&gt;ブートローダーの設定（GRUBの場合）&lt;/h1&gt;
sudo grub-mkconfig -o /boot/grub/grub.cfg
</code></pre>
再起動時にGRUBメニューで「Advanced options for Arch Linux」からLTSカーネルを選択できます。</p>
<h3>例3：手動でmkinitcpioを再実行</h3>
<pre><code class="language-bash">&lt;h1&gt;すべてのカーネルに対してinitramfsを再生成&lt;/h1&gt;
sudo mkinitcpio -P
<p>&lt;h1&gt;特定のカーネルのみ再生成（例：linux）&lt;/h1&gt;
sudo mkinitcpio -p linux
</code></pre>
<code>-P</code>オプションは<code>/etc/mkinitcpio.d/</code>に定義されたすべてのカーネルプリセットを処理します。</p>
<h3>例4：fallbackイメージを使用した起動</h3>
システムが起動しない場合、ブートローダーで<code>initramfs-linux-fallback.img</code>を選択して起動します。その後、問題を診断します。
<h2>よくある誤解</h2>
<h3>誤解1：「カーネルアップデート後は必ずmkinitcpioを手動で実行しなければならない」</h3>
実際には、<code>pacman</code>のフックにより自動実行されます。ただし、<code>/etc/mkinitcpio.conf</code>を変更した場合や、特定の状況では手動実行が必要です。
<h3>誤解2：「linux-ltsは古くて使えない」</h3>
<code>linux-lts</code>は最新の安定版ではありませんが、長期サポートによりセキュリティパッチが提供され続けます。最新機能が必要ないサーバーなどで推奨されます。
<h3>誤解3：「fallbackイメージは常に必要」</h3>
<code>fallback</code>イメージは予備であり、通常は不要です。ただし、カスタムカーネルや特殊なハードウェア構成では役立つことがあります。
<h3>誤解4：「pacman -Syuだけで全てが完了する」</h3>
カーネルアップデート後は再起動が必要です。また、ブートローダーの設定が自動更新されない場合があるため、必要に応じて<code>grub-mkconfig</code>などを実行します。
