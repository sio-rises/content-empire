2:T197f,<h2>はじめに</h2>
<p>Arch Linuxは最小限の構成から自分好みにカスタマイズできるディストリビューションですが、システム管理の要となるのが<strong>systemd</strong>です。本記事では、systemdの基本的な概念から、サービスの起動・停止・自動起動設定、ログの確認方法までを実践的に解説します。</p>
<hr>
<h2>systemdとは？</h2>
<h3>定義</h3>
systemdはLinuxの<strong>initシステム</strong>（システム起動時に最初に実行されるプロセス）であり、サービスの管理や依存関係の解決、並列起動などを担います。Arch Linuxではデフォルトで採用されています。
<h3>背景</h3>
従来のSysVinitやBSDスタイルのinitは直列処理が中心で、起動が遅く依存関係の記述が煩雑でした。systemdは<strong>ユニット</strong>という概念でサービスやマウントなどを統一的に管理し、ソケットやD-Busによる<strong>アクティベーション</strong>で効率的な起動を実現します。
<hr>
<h2>systemdの仕組み</h2>
<h3>ユニットとは</h3>
systemdが管理するリソースはすべて<strong>ユニット</strong>として定義されます。代表的なユニットタイプは以下の通りです。
<li><code>.service</code> : デーモンやサービスの定義</li>
<li><code>.socket</code> : ソケットアクティベーション</li>
<li><code>.target</code> : 複数ユニットのグループ（runlevel相当）</li>
<li><code>.timer</code> : 定期実行の定義（cron代替）</li>
<p>ユニットファイルは <code>/usr/lib/systemd/system/</code>（パッケージ提供）や <code>/etc/systemd/system/</code>（ユーザー設定）に配置されます。</p>
<h3>systemctlの基本コマンド</h3>
サービスの操作は <strong><code>systemctl</code></strong> コマンドで行います。
<p>#### サービスの状態確認
<pre><code class="language-bash">&lt;h1&gt;すべてのユニットの状態を表示&lt;/h1&gt;
systemctl list-units --type=service</p>
<p>&lt;h1&gt;特定のサービスの状態を表示（例: sshd）&lt;/h1&gt;
systemctl status sshd.service
</code></pre>
出力例では「active (running)」「enabled」「disabled」などが確認できます。</p>
<p>#### サービスの起動・停止・再起動
<pre><code class="language-bash">&lt;h1&gt;起動&lt;/h1&gt;
sudo systemctl start sshd.service</p>
<p>&lt;h1&gt;停止&lt;/h1&gt;
sudo systemctl stop sshd.service</p>
<p>&lt;h1&gt;再起動&lt;/h1&gt;
sudo systemctl restart sshd.service</p>
<p>&lt;h1&gt;設定ファイルを再読み込み（一部のサービス）&lt;/h1&gt;
sudo systemctl reload sshd.service
</code></pre></p>
<p>#### 自動起動の設定
<pre><code class="language-bash">&lt;h1&gt;自動起動を有効化&lt;/h1&gt;
sudo systemctl enable sshd.service</p>
<p>&lt;h1&gt;自動起動を無効化&lt;/h1&gt;
sudo systemctl disable sshd.service</p>
<p>&lt;h1&gt;自動起動の有効状態を確認&lt;/h1&gt;
systemctl is-enabled sshd.service
</code></pre>
<code>enable</code> は <code>/etc/systemd/system/</code> にシンボリックリンクを作成します。</p>
<p>#### 即時起動＋自動起動の両方を設定
<pre><code class="language-bash">sudo systemctl enable --now sshd.service
</code></pre>
このコマンドで有効化と即時起動を同時に行えます。</p>
<hr>
<h2>実用例：SSHサーバーの管理</h2>
<h3>1. SSHサーバーのインストールと起動</h3>
<pre><code class="language-bash">&lt;h1&gt;opensshパッケージをインストール&lt;/h1&gt;
sudo pacman -S openssh
<p>&lt;h1&gt;サービスを起動&lt;/h1&gt;
sudo systemctl start sshd.service</p>
<p>&lt;h1&gt;自動起動を有効化&lt;/h1&gt;
sudo systemctl enable sshd.service
</code></pre></p>
<h3>2. 状態確認とトラブルシューティング</h3>
<pre><code class="language-bash">&lt;h1&gt;状態確認&lt;/h1&gt;
systemctl status sshd.service
<p>&lt;h1&gt;ログを確認（-u でユニット指定）&lt;/h1&gt;
journalctl -u sshd.service -e
</code></pre>
<code>journalctl</code> はsystemdのログ管理ツール。<code>-e</code> で最新のログを表示します。</p>
<h3>3. 設定変更後の再読み込み</h3>
<code>/etc/ssh/sshd_config</code> を編集したら：
<pre><code class="language-bash">sudo systemctl reload sshd.service
</code></pre>
<code>restart</code> と異なり、<code>reload</code> は接続を切断せずに設定を再読み込みします。
<hr>
<h2>よくある誤解</h2>
<h3>誤解1: systemctl enable だけでサービスが起動する</h3>
<code>enable</code> は自動起動の設定のみで、即時起動はしません。起動するには <code>start</code> または <code>enable --now</code> を使います。
<h3>誤解2: サービス名の拡張子は必須</h3>
<code>systemctl status sshd</code> のように拡張子を省略できます。ただし <code>.service</code> 以外のユニット（<code>.socket</code> など）は指定が必要な場合があります。
<h3>誤解3: journalctl は常に全ログを表示する</h3>
<code>journalctl</code> はデフォルトで現在のブートのログのみ表示します。過去のログは <code>journalctl --list-boots</code> で確認し、<code>-b -1</code> などで指定します。
<h3>誤解4: systemdは複雑すぎる</h3>
確かに従来のinitより学習曲線はありますが、一度慣れれば統一的な管理が可能です。<code>man systemd.service</code> や <code>man systemctl</code> で詳細を確認しましょう。
