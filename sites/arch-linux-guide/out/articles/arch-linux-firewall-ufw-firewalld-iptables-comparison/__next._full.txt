11:T223f,<h2>はじめに</h2>
<p>Arch Linuxは軽量でカスタマイズ性の高いディストリビューションですが、セキュリティは自力で設定する必要があります。特にファイアウォールは基本中の基本です。本記事では、Arch Linuxで利用可能な主要なファイアウォールツール「ufw」「firewalld」「iptables」を比較し、それぞれの特徴や使い方を解説します。</p>
<h2>各ツールの概要</h2>
<h3>iptables</h3>
iptablesはLinuxカーネルのNetfilterフレームワークを操作する伝統的なファイアウォールツールです。ルールベースでパケットフィルタリングを行い、非常に細かい制御が可能です。Arch Linuxではデフォルトでインストールされており、<code>iptables</code>パッケージで提供されます。設定はコマンドラインから直接ルールを追加するか、<code>iptables-save</code>/<code>iptables-restore</code>で永続化します。
<h3>ufw (Uncomplicated Firewall)</h3>
ufwはiptablesのフロントエンドとして動作する、シンプルなファイアウォール管理ツールです。Ubuntuで標準採用されており、初心者にも扱いやすいコマンド体系が特徴。Arch Linuxでは<code>ufw</code>パッケージをインストールして使用します。
<h3>firewalld</h3>
firewalldは動的ファイアウォール管理デーモンで、ゾーンとサービス単位でルールを管理します。FedoraやRHEL系で標準ですが、Arch Linuxでも<code>firewalld</code>パッケージで利用可能。D-Bus経由で設定を動的に変更でき、NetworkManagerとの統合も強力です。

<h2>長所と短所</h2>
<h3>iptables</h3>
<strong>長所</strong>
<li>細かい制御が可能（特定のポート、IP、プロトコル、ステートフルルールなど）</li>
<li>軽量で依存関係が少ない</li>
<li>シェルスクリプトで自動化しやすい</li>
<li>他のツールの基盤として動作</li>
<p><strong>短所</strong>
<li>ルール記述が複雑で学習コストが高い</li>
<li>動的な変更が難しく、再起動でリセットされる（永続化には別途設定が必要）</li>
<li>ルールの順序管理が煩雑</li></p>
<h3>ufw</h3>
<strong>長所</strong>
<li>コマンドが直感的で覚えやすい（<code>ufw allow 22/tcp</code>など）</li>
<li>デフォルトで拒否ポリシーが適用されるため、すぐに安全</li>
<li>初心者でも簡単に設定可能</li>
<p><strong>短所</strong>
<li>iptablesの全機能を使えない（高度なルールは直接iptablesを編集する必要あり）</li>
<li>大量のルールがあると管理が煩雑になる</li>
<li>ログ機能が限定的</li></p>
<h3>firewalld</h3>
<strong>長所</strong>
<li>ゾーン概念により、ネットワークインターフェースごとに異なるポリシーを適用できる</li>
<li>動的な設定変更が可能で、Runtime/Permanentを分けて管理</li>
<li>NetworkManagerと統合され、ネットワーク変更時に自動でゾーン切り替え</li>
<li>サービス定義が豊富で、<code>firewall-cmd --add-service=http</code>のように簡単</li>
<p><strong>短所</strong>
<li>ゾーンやサービスの概念に慣れが必要</li>
<li>iptablesに比べてオーバーヘッドがある</li>
<li>複雑なルール（カスタムモジュールなど）は直接iptablesを併用する必要がある</li></p>
<h2>おすすめの選び方</h2>
<h3>初心者・簡単にセキュリティを高めたい場合 → ufw</h3>
コマンドが少なく、すぐに使い始められます。デフォルト拒否のポリシーで安全。SSHやWebサーバーなど、必要なサービスだけ許可する設定が簡単です。
<h3>ネットワーク環境が頻繁に変わる場合 → firewalld</h3>
ノートPCなど、異なるネットワークに接続する場合に便利。ゾーン（home, public, workなど）を自動切り替えでき、各環境に応じたルールを適用できます。
<h3>サーバー運用・細かい制御が必要な場合 → iptables</h3>
軽量で、カスタムルールを自由に記述できます。特定のIPのみ許可、レート制限、ログ出力など、高度な設定が必要なサーバー向け。
<h3>中級者・バランス重視 → firewalld</h3>
ゾーン機能を活かしつつ、iptablesの直接編集も併用することで、柔軟性と管理容易性を両立できます。
<h2>インストールと基本設定例</h2>
<h3>iptablesの基本設定</h3>
<pre><code class="language-bash">&lt;h1&gt;インストール（通常はデフォルトでインストール済み）&lt;/h1&gt;
sudo pacman -S iptables
<p>&lt;h1&gt;現在のルールを表示&lt;/h1&gt;
sudo iptables -L -n -v</p>
<p>&lt;h1&gt;SSH（22/tcp）を許可するルールを追加&lt;/h1&gt;
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT</p>
<p>&lt;h1&gt;デフォルトポリシーをDROPに設定（注意：SSH許可後に実行）&lt;/h1&gt;
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT</p>
<p>&lt;h1&gt;ルールを永続化&lt;/h1&gt;
sudo iptables-save &gt; /etc/iptables/iptables.rules
sudo systemctl enable iptables.service
sudo systemctl start iptables.service
</code></pre></p>
<h3>ufwの基本設定</h3>
<pre><code class="language-bash">&lt;h1&gt;インストール&lt;/h1&gt;
sudo pacman -S ufw
<p>&lt;h1&gt;デフォルトポリシーを設定（拒否）&lt;/h1&gt;
sudo ufw default deny</p>
<p>&lt;h1&gt;SSHを許可&lt;/h1&gt;
sudo ufw allow ssh</p>
<p>&lt;h1&gt;ファイアウォールを有効化&lt;/h1&gt;
sudo ufw enable</p>
<p>&lt;h1&gt;ステータス確認&lt;/h1&gt;
sudo ufw status verbose
</code></pre></p>
<h3>firewalldの基本設定</h3>
<pre><code class="language-bash">&lt;h1&gt;インストール&lt;/h1&gt;
sudo pacman -S firewalld
<p>&lt;h1&gt;サービスを有効化・起動&lt;/h1&gt;
sudo systemctl enable firewalld.service
sudo systemctl start firewalld.service</p>
<p>&lt;h1&gt;デフォルトゾーンを確認&lt;/h1&gt;
sudo firewall-cmd --get-default-zone</p>
<p>&lt;h1&gt;SSHサービスを許可&lt;/h1&gt;
sudo firewall-cmd --add-service=ssh --permanent
sudo firewall-cmd --reload</p>
<p>&lt;h1&gt;ゾーンを変更（例：publicからhomeへ）&lt;/h1&gt;
sudo firewall-cmd --set-default-zone=home
</code></pre></p>
