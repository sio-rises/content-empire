2:T1c4d,<h2>pacmanとは？</h2>
<p><strong>pacman</strong>（Package Managerの略）は、Arch Linuxの公式パッケージ管理ツールです。バイナリパッケージのインストール・更新・削除を効率的に行い、依存関係を自動解決します。初心者から上級者まで、Arch Linuxを使う上で必須のコマンドです。</p>
<h2>なぜpacmanが重要なのか？</h2>
<p>Arch Linuxはローリングリリースモデルを採用しており、システム全体を常に最新に保つためにパッケージ管理が頻繁に発生します。pacmanはシンプルながら強力で、一度覚えれば素早く操作できます。また、AUR（Arch User Repository）との連携も可能で、コミュニティが提供するパッケージも利用できます。</p>
<h2>pacmanの仕組み</h2>
<p>pacmanは、公式リポジトリ（core, extra, community, multilib）からパッケージをダウンロードし、<code>.pkg.tar.zst</code>形式のアーカイブを展開してインストールします。データベースは<code>/var/lib/pacman/</code>に保存され、パッケージ情報や依存関係を管理します。設定ファイルは<code>/etc/pacman.conf</code>です。</p>
<h2>実用的なコマンドチートシート</h2>
<h3>1. パッケージのインストール</h3>
<pre><code class="language-bash">&lt;h1&gt;単一パッケージのインストール&lt;/h1&gt;
sudo pacman -S パッケージ名
<p>&lt;h1&gt;複数パッケージを同時にインストール&lt;/h1&gt;
sudo pacman -S パッケージ1 パッケージ2</p>
<p>&lt;h1&gt;グループからインストール（例: xfce4）&lt;/h1&gt;
sudo pacman -S xfce4-group
</code></pre></p>
<p><code>-S</code>はsyncの略で、リポジトリと同期してインストールします。</p>
<h3>2. パッケージの更新</h3>
<pre><code class="language-bash">&lt;h1&gt;リポジトリデータベースの更新&lt;/h1&gt;
sudo pacman -Sy
<p>&lt;h1&gt;全パッケージの更新（推奨）&lt;/h1&gt;
sudo pacman -Syu</p>
<p>&lt;h1&gt;特定パッケージのみ更新&lt;/h1&gt;
sudo pacman -S パッケージ名
</code></pre></p>
<p><code>-Syu</code>はデータベース更新と全パッケージ更新を同時に行います。<code>-Sy</code>だけでは部分更新になり、依存関係が壊れる可能性があるため、常に<code>-Syu</code>を使いましょう。</p>
<h3>3. パッケージの削除</h3>
<pre><code class="language-bash">&lt;h1&gt;パッケージのみ削除（設定ファイルは残る）&lt;/h1&gt;
sudo pacman -R パッケージ名
<p>&lt;h1&gt;パッケージとその依存関係を削除（他のパッケージで使われていないもののみ）&lt;/h1&gt;
sudo pacman -Rs パッケージ名</p>
<p>&lt;h1&gt;パッケージと設定ファイルも削除&lt;/h1&gt;
sudo pacman -Rns パッケージ名
</code></pre></p>
<p><code>-R</code>はremoveの略。<code>-Rs</code>は依存関係も削除するので、不要なパッケージをきれいに掃除できます。</p>
<h3>4. パッケージの検索</h3>
<pre><code class="language-bash">&lt;h1&gt;パッケージ名と説明で検索&lt;/h1&gt;
pacman -Ss キーワード
<p>&lt;h1&gt;インストール済みパッケージを検索&lt;/h1&gt;
pacman -Qs キーワード</p>
<p>&lt;h1&gt;特定のファイルがどのパッケージに属するか調べる&lt;/h1&gt;
pacman -Qo /path/to/file
</code></pre></p>
<p><code>-Ss</code>はリポジトリ内を検索、<code>-Qs</code>はローカルデータベースを検索します。<code>-Qo</code>はファイルの所有者を特定するのに便利です。</p>
<h3>5. インストール済みパッケージの情報表示</h3>
<pre><code class="language-bash">&lt;h1&gt;インストール済みパッケージの一覧&lt;/h1&gt;
pacman -Q
<p>&lt;h1&gt;明示的にインストールしたパッケージのみ表示&lt;/h1&gt;
pacman -Qe</p>
<p>&lt;h1&gt;パッケージの詳細情報&lt;/h1&gt;
pacman -Qi パッケージ名</p>
<p>&lt;h1&gt;パッケージに含まれるファイル一覧&lt;/h1&gt;
pacman -Ql パッケージ名
</code></pre></p>
<p><code>-Q</code>はqueryの略。<code>-Qe</code>は依存関係で自動インストールされたパッケージを除外します。</p>
<h3>6. キャッシュのクリーンアップ</h3>
<pre><code class="language-bash">&lt;h1&gt;ダウンロードしたパッケージキャッシュを削除（最新以外は削除）&lt;/h1&gt;
sudo pacman -Sc
<p>&lt;h1&gt;全キャッシュを削除&lt;/h1&gt;
sudo pacman -Scc
</code></pre></p>
<p>キャッシュは<code>/var/cache/pacman/pkg/</code>に保存されます。<code>-Sc</code>は古いバージョンのみ削除し、<code>-Scc</code>は全て削除します。</p>
<h3>7. トラブルシューティング</h3>
<pre><code class="language-bash">&lt;h1&gt;データベースがロックされている場合の解除&lt;/h1&gt;
sudo rm /var/lib/pacman/db.lck
<p>&lt;h1&gt;パッケージデータベースの整合性チェック&lt;/h1&gt;
sudo pacman -Dk</p>
<p>&lt;h1&gt;破損したパッケージの再インストール&lt;/h1&gt;
sudo pacman -Syu --force パッケージ名
</code></pre></p>
<p><code>db.lck</code>はpacman実行中に作成されるロックファイルです。異常終了した場合に削除して復旧します。</p>
<h2>よくある誤解と注意点</h2>
<h3>❌ <code>pacman -Sy</code> を頻繁に実行する</h3>
<code>-Sy</code>だけだとデータベースのみ更新し、パッケージは古いままです。後で<code>pacman -S パッケージ</code>を実行すると、依存関係が不整合になる可能性があります。必ず<code>-Syu</code>で更新しましょう。
<h3>❌ <code>pacman -R</code> で依存関係を無視して削除する</h3>
<code>-R</code>はパッケージのみ削除し、依存関係は残ります。不要な依存パッケージが溜まる原因になるので、<code>-Rs</code>を使う習慣をつけましょう。
<h3>❌ キャッシュを削除しすぎる</h3>
<code>-Scc</code>で全キャッシュを削除すると、ダウングレードが必要な場合に再ダウンロードが必要になります。通常は<code>-Sc</code>で十分です。
<h3>❌ AURパッケージをpacmanで直接インストールする</h3>
AURのパッケージはpacmanでは管理できません。<code>yay</code>や<code>paru</code>などのAURヘルパーを使うか、手動でPKGBUILDからビルドします。
