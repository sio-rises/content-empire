<h2>Arch Linux インストール後 やることリスト：初期設定・必須設定を完全網羅</h2>
<p>Arch Linuxのインストールが完了したら、次にやるべきことは<strong>初期設定</strong>です。この記事では、インストール直後に行うべき必須設定を、初心者にもわかりやすく解説します。</p>
<h3>1. ネットワークの設定</h3>
<p>インストール直後はネットワークが使えない場合があります。有線LANの場合はDHCPで自動取得、無線LANの場合は<code>iwd</code>や<code>NetworkManager</code>を使って接続します。</p>
<h4>有線LANの場合</h4>
<code>systemd-networkd</code>と<code>systemd-resolved</code>を有効化します。
<pre><code class="language-bash">sudo systemctl enable --now systemd-networkd systemd-resolved
</code></pre>
<h4>無線LANの場合</h4>
<code>iwd</code>をインストールして有効化します。
<pre><code class="language-bash">sudo pacman -S iwd
sudo systemctl enable --now iwd
</code></pre>
<p>その後、<code>iwctl</code>で接続します。</p>
<pre><code class="language-bash">iwctl
[iwd]# device list
[iwd]# station wlan0 scan
[iwd]# station wlan0 connect [SSID]
</code></pre>
<p><code>NetworkManager</code>を使う場合は、<code>networkmanager</code>パッケージをインストールして有効化します。</p>
<pre><code class="language-bash">sudo pacman -S networkmanager
sudo systemctl enable --now NetworkManager
</code></pre>
<h3>2. パッケージのアップデートとミラー設定</h3>
<p>インストール直後はパッケージが古い可能性があるため、まずはアップデートします。</p>
<pre><code class="language-bash">sudo pacman -Syu
</code></pre>
<p>ミラーサーバーを最適化するには、<code>reflector</code>を使います。</p>
<pre><code class="language-bash">sudo pacman -S reflector
sudo reflector --latest 5 --protocol https --sort rate --save /etc/pacman.d/mirrorlist
</code></pre>
<h3>3. ユーザー作成とsudo設定</h3>
<p>インストール時にrootユーザーしか作成していない場合は、一般ユーザーを作成します。</p>
<pre><code class="language-bash">useradd -m -G wheel -s /bin/bash ユーザー名
passwd ユーザー名
</code></pre>
<p>sudoを使えるようにするため、<code>sudo</code>パッケージをインストールし、visudoで<code>%wheel ALL=(ALL:ALL) ALL</code>のコメントを外します。</p>
<pre><code class="language-bash">sudo pacman -S sudo
EDITOR=vim sudo visudo
</code></pre>
<h3>4. デスクトップ環境のインストール（任意）</h3>
<p>GUIを使いたい場合は、デスクトップ環境をインストールします。代表的なものとして、KDE Plasma、GNOME、Xfceなどがあります。</p>
<p>例：KDE Plasma</p>
<pre><code class="language-bash">sudo pacman -S plasma-meta kde-applications-meta
sudo systemctl enable sddm
</code></pre>
<h3>5. AURヘルパーのインストール</h3>
<p>AUR（Arch User Repository）を使うために、<code>yay</code>や<code>paru</code>をインストールします。</p>
<pre><code class="language-bash">sudo pacman -S --needed base-devel git
git clone https://aur.archlinux.org/yay.git
cd yay
makepkg -si
</code></pre>
<h3>6. 日本語入力の設定</h3>
<p>日本語を入力できるようにするため、<code>fcitx5</code>と日本語入力エンジン<code>mozc</code>をインストールします。</p>
<pre><code class="language-bash">sudo pacman -S fcitx5 fcitx5-mozc fcitx5-configtool
</code></pre>
<p>環境変数を設定します。</p>
<pre><code class="language-bash">echo 'export GTK_IM_MODULE=fcitx' &gt;&gt; ~/.xprofile
echo 'export QT_IM_MODULE=fcitx' &gt;&gt; ~/.xprofile
echo 'export XMODIFIERS=@im=fcitx' &gt;&gt; ~/.xprofile
</code></pre>
<h3>7. ファイアウォールの設定</h3>
<p>セキュリティのために、<code>ufw</code>や<code>firewalld</code>を設定します。簡単なのは<code>ufw</code>です。</p>
<pre><code class="language-bash">sudo pacman -S ufw
sudo ufw enable
sudo ufw allow 22/tcp  # SSHを許可する場合
</code></pre>
<h3>8. スワップファイルの作成（任意）</h3>
<p>スワップ領域がない場合、スワップファイルを作成します。</p>
<pre><code class="language-bash">sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
</code></pre>
<p><code>/etc/fstab</code>に追記します。</p>
<pre><code class="language-bash">echo '/swapfile none swap defaults 0 0' | sudo tee -a /etc/fstab
</code></pre>
<h3>9. よくある誤解</h3>
<h4>誤解1: <code>pacman -Syu</code>はいつでも安全</h4>
<p><strong>事実</strong>: カーネルやシステムライブラリの更新後は再起動が必要です。また、<code>pacman -Syu</code>は部分アップグレードを避けるため、必ずフルアップグレードを行います。</p>
<h4>誤解2: AURヘルパーは必須</h4>
<p><strong>事実</strong>: AURヘルパーは便利ですが、手動で<code>makepkg</code>を使うこともできます。セキュリティ上、信頼できるPKGBUILDのみを使用しましょう。</p>
<h4>誤解3: デスクトップ環境は必須</h4>
<p><strong>事実</strong>: Arch Linuxはターミナルベースでも十分に使えます。サーバー用途ならデスクトップ環境は不要です。</p>
<h3>まとめ</h3>
<p>以上が、Arch Linuxインストール後の初期設定の主要な項目です。ネットワーク設定、パッケージ管理、ユーザー設定、日本語入力、ファイアウォールなど、順を追って設定することで、快適なArch Linux環境が構築できます。</p>
