<h2>Arch Linux AURパッケージ管理入門：yayとparuの違いと使い方</h2>
<p>Arch Linuxの魅力の一つに、豊富なパッケージを提供するAUR（Arch User Repository）があります。しかし、初心者にとっては「AURって何？」「どうやって使うの？」と疑問に思うことも多いでしょう。この記事では、AURの基本から、代表的なAURヘルパーであるyayとparuの違い、実際の使い方までを解説します。</p>
<h3>AUR（Arch User Repository）とは？</h3>
<p>AURは、Archユーザーが作成したパッケージを共有するコミュニティリポジトリです。公式リポジトリには含まれていないソフトウェアも、AURを通じて簡単にインストールできます。</p>
<li><strong>定義</strong>: AURは、ユーザーが作成したPKGBUILD（ビルドスクリプト）を集めたリポジトリで、makepkgを使ってソースからパッケージをビルドする仕組みです。</li>
<li><strong>背景</strong>: Arch Linuxはシンプルさを重視しており、公式リポジトリには厳選されたパッケージしか含まれません。しかし、ユーザーから「もっと多くのソフトウェアを簡単にインストールしたい」という要望が高まり、AURが誕生しました。これにより、コミュニティが自由にパッケージを提供できるようになり、Archのエコシステムが拡大しました。</li>
<li><strong>仕組み</strong>: AURの各パッケージはPKGBUILDというスクリプトで構成されています。PKGBUILDには、ソースコードの入手先、ビルド手順、依存関係などが記述されています。ユーザーはこのPKGBUILDをダウンロードし、<code>makepkg</code>コマンドでビルド・インストールします。ただし、手動で行うのは面倒なため、AURヘルパーが自動化してくれます。</li>
<h3>AURヘルパー：yayとparu</h3>
<p>AURヘルパーは、AURパッケージの検索、ダウンロード、ビルド、インストールを自動化するツールです。代表的なものにyayとparuがあります。</p>
<h4>yay（Yet Another Yogurt）</h4>
<li><strong>特徴</strong>: 長年使われてきた定番のAURヘルパー。シンプルで安定しており、多くのユーザーに支持されています。</li>
<li><strong>インストール</strong>:</li>
  <pre><code class="language-bash">  sudo pacman -S --needed git base-devel
  git clone https://aur.archlinux.org/yay.git
  cd yay
  makepkg -si
  </code></pre>
<li><strong>基本的な使い方</strong>:</li>
  - パッケージの検索: <code>yay パッケージ名</code>
  - パッケージのインストール: <code>yay -S パッケージ名</code>
  - システム全体の更新（AUR含む）: <code>yay -Syu</code>
<h4>paru（パル）</h4>
<li><strong>特徴</strong>: yayの後継として開発された、Rust製の高速なAURヘルパー。並列ビルドやカラフルな出力、pacmanとの統合が強化されています。</li>
<li><strong>インストール</strong>:</li>
  <pre><code class="language-bash">  sudo pacman -S --needed base-devel
  git clone https://aur.archlinux.org/paru.git
  cd paru
  makepkg -si
  </code></pre>
<li><strong>基本的な使い方</strong>:</li>
  - パッケージの検索: <code>paru パッケージ名</code>
  - パッケージのインストール: <code>paru -S パッケージ名</code>
  - システム全体の更新（AUR含む）: <code>paru -Syu</code>
<h4>yayとparuの違い</h4>
<table><thead><tr><th>機能</th><th>yay</th><th>paru</th></tr></thead><tbody><tr><td>言語</td><td>Go</td><td>Rust</td></tr>
<tr><td>並列ビルド</td><td>なし（順次）</td><td>あり（高速）</td></tr>
<tr><td>設定ファイル</td><td>~/.config/yay/config.json</td><td>/etc/paru.conf, ~/.config/paru/paru.conf</td></tr>
<tr><td>出力の色付け</td><td>限定的</td><td>カラフル</td></tr>
<tr><td>pacmanラッパー</td><td>あり</td><td>あり（より統合）</td></tr>
<tr><td>開発の活発さ</td><td>安定しているが更新は緩やか</td><td>活発に開発中</td></tr></tbody></table>
初心者には、ドキュメントが豊富でコミュニティも大きいyayをおすすめします。一方、速度やモダンな機能を重視するならparuが適しています。
<h3>実用例：yayとparuでパッケージをインストール</h3>
<p>ここでは、例として「google-chrome」をインストールする手順を示します。</p>
<h4>yayの場合</h4>
<li>パッケージを検索:</li>
   <pre><code class="language-bash">   yay google-chrome
   </code></pre>
   すると、AUR内の関連パッケージ一覧が表示されます。
<li>インストール:</li>
   <pre><code class="language-bash">   yay -S google-chrome
   </code></pre>
   依存関係の確認後、ビルドが開始され、自動的にインストールされます。
<h4>paruの場合</h4>
<li>パッケージを検索:</li>
   <pre><code class="language-bash">   paru google-chrome
   </code></pre>
<li>インストール:</li>
   <pre><code class="language-bash">   paru -S google-chrome
   </code></pre>
   paruは並列ビルドにより、複数の依存パッケージを同時にビルドするため、yayより高速な場合があります。
<h3>よくある誤解</h3>
<li><strong>AURは公式リポジトリと同じくらい安全？</strong></li>
  いいえ。AURのパッケージはコミュニティによって管理されており、公式のような厳格な審査はありません。PKGBUILDを確認するなど、自己責任で使用する必要があります。
<li><strong>yayやparuはpacmanの代わりになる？</strong></li>
  部分的にはそうです。yayやparuはpacmanのラッパーとして動作し、pacmanのコマンド（<code>-S</code>, <code>-R</code>, <code>-Q</code>など）をそのまま使えます。ただし、完全に置き換えるわけではなく、pacman自体もインストールされています。
<li><strong>AURヘルパーを使わなくてもAURは使える？</strong></li>
  はい。手動でPKGBUILDをダウンロードし、<code>makepkg -si</code>でインストールできます。ただし、依存関係の解決や更新が面倒なため、ヘルパーの使用が推奨されます。
