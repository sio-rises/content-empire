2:T17b8,<h2>はじめに</h2>
<p>この記事では、Arch Linuxを手動でインストールする手順を解説します。自動インストールスクリプトを使わず、一つ一つコマンドを実行することで、Linuxのブートプロセスやパーティション構成への理解が深まります。対象読者は、Linuxの基本的なコマンド操作（ls, cd, viなど）を知っている初心者〜中級者です。仮想環境（VirtualBoxなど）で実践することを推奨します。</p>
<h2>前提条件</h2>
<li>Arch LinuxのISOイメージをダウンロードし、USBメモリなどに書き込んでいること</li>
<li>UEFIモードで起動できる環境（最近のPCなら対応）</li>
<li>インターネット接続（有線推奨）</li>
<h2>手順</h2>
<h3>ステップ1：ライブ環境の起動と確認</h3>
<li>ISOから起動し、"Arch Linux install medium"を選択します。</li>
<li>起動後、以下のコマンドでUEFIモードか確認します。</li>
<pre><code class="language-bash">ls /sys/firmware/efi/efivars
</code></pre>
<p>ディレクトリが存在すればUEFIモードです。存在しない場合はBIOS/CSMモードなので、後の手順でパーティションをMBRにする必要があります。</p>
<h3>ステップ2：キーボードレイアウトの設定</h3>
<p>日本語キーボードの場合は以下を実行します。</p>
<pre><code class="language-bash">loadkeys jp106
</code></pre>
<h3>ステップ3：インターネット接続の確認</h3>
<pre><code class="language-bash">ping -c 3 archlinux.org
</code></pre>
<p>有線LANなら自動で接続されます。Wi-Fiの場合はiwctlコマンドを使います（詳細はArch Wiki参照）。</p>
<h3>ステップ4：ディスクのパーティショニング</h3>
<p>ここでは例として、/dev/sdaにUEFI+GPTでパーティションを作成します。</p>
<pre><code class="language-bash">fdisk /dev/sda
</code></pre>
<p>以下のように設定します。</p>
<li>パーティション1: 512MB, EFIシステムパーティション (タイプ: EFI System)</li>
<li>パーティション2: 4GB, スワップ (タイプ: Linux swap)</li>
<li>パーティション3: 残りすべて, ルート (タイプ: Linux filesystem)</li>
<p>変更を書き込んだら、各パーティションをフォーマットします。</p>
<pre><code class="language-bash">mkfs.fat -F32 /dev/sda1
mkswap /dev/sda2
mkfs.ext4 /dev/sda3
</code></pre>
<h3>ステップ5：ファイルシステムのマウント</h3>
<pre><code class="language-bash">mount /dev/sda3 /mnt
mkdir -p /mnt/boot &amp;&amp; mount /dev/sda1 /mnt/boot
swapon /dev/sda2
</code></pre>
<h3>ステップ6：ベースシステムのインストール</h3>
<pre><code class="language-bash">pacstrap -K /mnt base base-devel linux linux-firmware vim
</code></pre>
<h3>ステップ7：fstabの生成</h3>
<pre><code class="language-bash">genfstab -U /mnt &gt;&gt; /mnt/etc/fstab
</code></pre>
<h3>ステップ8：chrootしてシステム設定</h3>
<pre><code class="language-bash">arch-chroot /mnt
</code></pre>
<p>#### タイムゾーンの設定</p>
<pre><code class="language-bash">ln -sf /usr/share/zoneinfo/Asia/Tokyo /etc/localtime
hwclock --systohc
</code></pre>
<p>#### ロケールの設定</p>
<pre><code class="language-bash">vim /etc/locale.gen
&lt;h1&gt;ja_JP.UTF-8 UTF-8 のコメントを解除&lt;/h1&gt;
locale-gen
echo "LANG=ja_JP.UTF-8" &gt; /etc/locale.conf
</code></pre>
<p>#### ホスト名の設定</p>
<pre><code class="language-bash">echo "myarch" &gt; /etc/hostname
</code></pre>
<p>#### ネットワーク設定</p>
<pre><code class="language-bash">vim /etc/hosts
&lt;h1&gt;以下を追加&lt;/h1&gt;
127.0.0.1 localhost
::1 localhost
127.0.1.1 myarch.localdomain myarch
</code></pre>
<p>#### パスワード設定</p>
<pre><code class="language-bash">passwd
</code></pre>
<p>#### ブートローダーのインストール（GRUB）</p>
<pre><code class="language-bash">pacman -S grub efibootmgr
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg
</code></pre>
<h3>ステップ9：再起動</h3>
<pre><code class="language-bash">exit
umount -R /mnt
reboot
</code></pre>
<p>起動後、ユーザー名root、設定したパスワードでログインします。</p>
<h2>トラブルシューティング</h2>
<h3>エラー1：ブートローダーが認識されない</h3>
<p>UEFI設定でセキュアブートを無効にしてください。また、ブート順序でGRUBが先頭になっているか確認します。</p>
<h3>エラー2：ネットワークが使えない</h3>
<pre><code class="language-bash">ip link
</code></pre>
でインターフェース名を確認し、
<pre><code class="language-bash">systemctl enable --now dhcpcd
</code></pre>
<p>でDHCPクライアントを起動します。</p>
<h3>エラー3：パッケージのインストールに失敗する</h3>
<p>ミラーリストが古い可能性があります。</p>
<pre><code class="language-bash">pacman -Syy
</code></pre>
<p>で更新してから再試行します。</p>
