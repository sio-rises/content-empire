11:T1c49,<h2>はじめに</h2>
<p>この記事では、既にWindowsがインストールされている環境にArch Linuxを追加でインストールし、GRUBで起動を管理するデュアルブート環境の構築方法を解説します。UEFI + GPT構成を前提とし、EFIパーティションをWindowsと共有する方法を取ります。</p>
<h2>事前準備</h2>
<li><strong>Windowsのセットアップ</strong></li>
   - 高速スタートアップを無効にする（コントロールパネル → 電源オプション → 電源ボタンの動作 → 現在利用可能ではない設定を変更 → 高速スタートアップのチェックを外す）
   - BitLockerが有効な場合は一時的に無効にする
<li><strong>Arch Linuxのインストールメディアを作成</strong></li>
   - <a href="https://archlinux.org/download/" target="_blank" rel="noopener">Arch Linux公式サイト</a>からISOをダウンロード
   - RufusやddコマンドでUSBメモリに書き込む
<li><strong>ディスクの空き領域を確保</strong></li>
   - Windowsのディスク管理ツールで、既存のパーティションを縮小して未割り当て領域を作成（最低20GB推奨）
<h2>インストール手順</h2>
<h3>1. Arch Linuxのインストール</h3>
<p>USBメモリから起動し、以下の手順でインストールを進めます。</p>
<pre><code class="language-bash">&lt;h1&gt;キーボードレイアウトの設定（日本語キーボードの場合）&lt;/h1&gt;
loadkeys jp106
<p>&lt;h1&gt;UEFIモードか確認&lt;/h1&gt;
ls /sys/firmware/efi/efivars
&lt;h1&gt;空でなければUEFIモード&lt;/h1&gt;</p>
<p>&lt;h1&gt;インターネット接続の確認&lt;/h1&gt;
ping -c 3 google.com</p>
<p>&lt;h1&gt;パーティションの作成（例: /dev/sdaにインストール）&lt;/h1&gt;
&lt;h1&gt;既存のWindowsパーティションは変更しない&lt;/h1&gt;
&lt;h1&gt;未割り当て領域に以下のパーティションを作成&lt;/h1&gt;
fdisk /dev/sda
&lt;h1&gt;コマンド例:&lt;/h1&gt;
&lt;h1&gt;n (新しいパーティション)&lt;/h1&gt;
&lt;h1&gt;パーティションタイプ: EFI System (既存のEFIパーティションがある場合は作成しない)&lt;/h1&gt;
&lt;h1&gt;パーティションタイプ: Linux filesystem (ルート用)&lt;/h1&gt;
&lt;h1&gt;パーティションタイプ: Linux swap (スワップ用、必要に応じて)&lt;/h1&gt;</p>
<p>&lt;h1&gt;パーティションをフォーマット&lt;/h1&gt;
mkfs.fat -F32 /dev/sdaX   # EFIパーティション（既存の場合はフォーマットしない）
mkfs.ext4 /dev/sdaY       # ルートパーティション
mkswap /dev/sdaZ          # スワップパーティション
swapon /dev/sdaZ</p>
<p>&lt;h1&gt;マウント&lt;/h1&gt;
mount /dev/sdaY /mnt
mount --mkdir /dev/sdaX /mnt/boot  # EFIパーティションを/bootにマウント</p>
<p>&lt;h1&gt;ベースシステムのインストール&lt;/h1&gt;
pacstrap -K /mnt base base-devel linux linux-firmware vim grub efibootmgr networkmanager</p>
<p>&lt;h1&gt;fstabの生成&lt;/h1&gt;
genfstab -U /mnt &gt;&gt; /mnt/etc/fstab</p>
<p>&lt;h1&gt;chroot&lt;/h1&gt;
arch-chroot /mnt</p>
<p>&lt;h1&gt;タイムゾーンとロケールの設定&lt;/h1&gt;
ln -sf /usr/share/zoneinfo/Asia/Tokyo /etc/localtime
hwclock --systohc
echo "en_US.UTF-8 UTF-8" &gt;&gt; /etc/locale.gen
echo "ja_JP.UTF-8 UTF-8" &gt;&gt; /etc/locale.gen
locale-gen
echo "LANG=en_US.UTF-8" &gt; /etc/locale.conf</p>
<p>&lt;h1&gt;ホスト名の設定&lt;/h1&gt;
echo "archlinux" &gt; /etc/hostname</p>
<p>&lt;h1&gt;rootパスワードの設定&lt;/h1&gt;
passwd</p>
<p>&lt;h1&gt;ユーザーの作成（任意）&lt;/h1&gt;
useradd -m -G wheel -s /bin/bash username
passwd username</p>
<p>&lt;h1&gt;sudoの設定（wheelグループにsudo権限）&lt;/h1&gt;
echo "%wheel ALL=(ALL:ALL) ALL" &gt;&gt; /etc/sudoers</p>
<p>&lt;h1&gt;ネットワークマネージャーの有効化&lt;/h1&gt;
systemctl enable NetworkManager</p>
<p>&lt;h1&gt;GRUBのインストール（UEFI）&lt;/h1&gt;
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB</p>
<p>&lt;h1&gt;GRUB設定ファイルの生成&lt;/h1&gt;
grub-mkconfig -o /boot/grub/grub.cfg</p>
<p>&lt;h1&gt;終了と再起動&lt;/h1&gt;
exit
umount -R /mnt
reboot
</code></pre></p>
<h3>2. WindowsとArch Linuxのデュアルブート確認</h3>
<p>再起動後、UEFIブートメニューで「GRUB」を選択すると、GRUBメニューが表示されます。Windows Boot ManagerとArch Linuxの両方が表示されることを確認します。</p>
<h2>トラブルシューティング</h2>
<h3>WindowsがGRUBメニューに表示されない</h3>
<p>原因: os-proberが正しく動作していない可能性があります。</p>
<p>解決策:
<pre><code class="language-bash">&lt;h1&gt;os-proberをインストール&lt;/h1&gt;
sudo pacman -S os-prober</p>
<p>&lt;h1&gt;GRUB設定を再生成&lt;/h1&gt;
sudo grub-mkconfig -o /boot/grub/grub.cfg
</code></pre></p>
<h3>GRUBが起動しない / UEFI設定が消える</h3>
<p>原因: EFIパーティションのブートエントリが正しく設定されていない。</p>
<p>解決策:
<pre><code class="language-bash">&lt;h1&gt;Arch Linuxのインストールメディアから起動し、chrootする&lt;/h1&gt;
&lt;h1&gt;EFIパーティションをマウント&lt;/h1&gt;
mount /dev/sdaX /mnt/boot
arch-chroot /mnt</p>
<p>&lt;h1&gt;GRUBを再インストール&lt;/h1&gt;
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB --recheck
grub-mkconfig -o /boot/grub/grub.cfg
</code></pre></p>
<h3>Windowsが起動しない（GRUBから選択しても）</h3>
<p>原因: Windowsの高速スタートアップが有効、またはBitLockerが原因。</p>
<p>解決策:
<li>Windowsで高速スタートアップを無効にする。</li>
<li>BitLockerを一時的に解除する。</li></p>
<h3>時刻がずれる</h3>
<p>Windowsはローカル時刻、LinuxはUTCをデフォルトで使用するため、時刻がずれることがあります。</p>
<p>解決策: Linuxでローカル時刻を使用するように設定。
<pre><code class="language-bash">sudo timedatectl set-local-rtc 1
</code></pre></p>

<h2>まとめ</h2>
<p>この記事では、Arch LinuxとWindowsのデュアルブート環境をGRUB + EFIで構築する方法を解説しました。ポイントは以下の通りです。</p>
<li>Windowsの高速スタートアップを無効にしておく</li>
<li>EFIパーティションはWindowsと共有するため、フォーマットせずに/bootにマウント</li>
<li>GRUBのインストール後、<code>grub-mkconfig</code>でWindowsを自動検出</li>
<li>トラブル時はos-proberのインストールやGRUBの再インストールを試す</li>
