2:T24e8,<h2>はじめに</h2>
<p>Arch Linuxでネットワークを管理するには、<code>systemd-networkd</code>や<code>netctl</code>など複数の選択肢がありますが、初心者から上級者まで幅広く使われているのが<strong>NetworkManager</strong>です。NetworkManagerは、有線・無線・VPNなどを統一的に管理し、コマンドラインの<code>nmcli</code>やGUIツールから簡単に設定できます。</p>
<p>この記事では、Arch LinuxにNetworkManagerをインストールし、<code>nmcli</code>を使った基本的なネットワーク設定、さらにGUIツールの導入方法をステップバイステップで解説します。トラブルシューティングもまとめているので、つまずいたときの参考にしてください。</p>
<h2>1. NetworkManagerのインストールと基本設定</h2>
<h3>1.1 インストール</h3>
<p>まず、NetworkManagerと必要な依存パッケージをインストールします。ターミナルを開いて以下を実行してください。</p>
<pre><code class="language-bash">sudo pacman -S networkmanager
</code></pre>
<p>無線接続を使う場合は、<code>wpa_supplicant</code>も一緒にインストールされます。追加で無線ファームウェアが必要な場合があります（例：<code>linux-firmware</code>）。</p>
<h3>1.2 サービスの有効化と起動</h3>
<p>インストール後、NetworkManagerサービスを有効にして起動します。</p>
<pre><code class="language-bash">sudo systemctl enable NetworkManager
sudo systemctl start NetworkManager
</code></pre>
<p>これで、NetworkManagerがシステム起動時に自動起動し、現在も動作している状態になります。</p>
<h3>1.3 既存のネットワーク管理ツールとの競合を避ける</h3>
<p>Arch Linuxでは、初期状態で<code>systemd-networkd</code>や<code>netctl</code>が有効になっている場合があります。これらが競合する可能性があるので、無効化しておきましょう。</p>
<pre><code class="language-bash">sudo systemctl disable systemd-networkd
sudo systemctl stop systemd-networkd
sudo systemctl disable netctl
sudo systemctl stop netctl
</code></pre>
<p>ただし、<code>systemd-resolved</code>は併用可能な場合もあります。必要に応じて設定を調整してください。</p>
<h2>2. nmcliを使ったネットワーク設定</h2>
<p><code>nmcli</code>はNetworkManagerのコマンドラインツールです。リモートサーバーやターミナル環境で重宝します。</p>
<h3>2.1 デバイスの確認</h3>
<p>まず、利用可能なネットワークデバイス（インターフェース）を確認します。</p>
<pre><code class="language-bash">nmcli device status
</code></pre>
<p>出力例：
<pre><code class="language-">DEVICE  TYPE      STATE         CONNECTION
eth0    ethernet  connected     Wired connection 1
wlan0   wifi      disconnected  --
</code></pre></p>
<h3>2.2 Wi-Fiに接続する</h3>
<p>Wi-Fiに接続するには、まずアクセスポイントをスキャンします。</p>
<pre><code class="language-bash">nmcli device wifi list
</code></pre>
<p>接続したいSSIDがあることを確認したら、以下のコマンドで接続します。</p>
<pre><code class="language-bash">nmcli device wifi connect "SSID名" password "パスワード"
</code></pre>
<p>成功すると、接続プロファイルが自動的に作成され、次回以降は自動接続されます。</p>
<h3>2.3 有線接続の設定</h3>
<p>有線接続は通常、ケーブルを挿すだけで自動的に接続されます。手動で設定する場合は、次のように接続プロファイルを作成します。</p>
<pre><code class="language-bash">nmcli connection add type ethernet con-name "MyWired" ifname eth0 ipv4.method manual ipv4.addresses 192.168.1.100/24 ipv4.gateway 192.168.1.1 ipv4.dns 8.8.8.8
</code></pre>
<p><code>con-name</code>はプロファイル名、<code>ifname</code>はデバイス名です。DHCPを使う場合は<code>ipv4.method auto</code>とします。</p>
<h3>2.4 接続の管理</h3>
<p>接続プロファイル一覧を表示：</p>
<pre><code class="language-bash">nmcli connection show
</code></pre>
<p>特定のプロファイルをアクティブにする：</p>
<pre><code class="language-bash">nmcli connection up "プロファイル名"
</code></pre>
<p>切断する：</p>
<pre><code class="language-bash">nmcli connection down "プロファイル名"
</code></pre>
<p>プロファイルを削除する：</p>
<pre><code class="language-bash">nmcli connection delete "プロファイル名"
</code></pre>
<h2>3. GUIツールの導入</h2>
<p>デスクトップ環境を使っているなら、GUIツールをインストールすると便利です。</p>
<h3>3.1 nm-connection-editor (GTK)</h3>
<p>GNOMEなどのGTKベース環境で使える設定エディタです。</p>
<pre><code class="language-bash">sudo pacman -S network-manager-applet
</code></pre>
<p>インストール後、アプリケーションメニューから「ネットワーク設定」または<code>nm-connection-editor</code>コマンドで起動できます。</p>
<h3>3.2 plasma-nm (KDE)</h3>
<p>KDE Plasma用のアプレットです。</p>
<pre><code class="language-bash">sudo pacman -S plasma-nm
</code></pre>
<p>システムトレイからネットワーク設定が可能になります。</p>
<h3>3.3 その他のフロントエンド</h3>
<li><code>nmcli</code>の簡易的なTUI（テキストユーザーインターフェース）として<code>nmtui</code>も利用できます。</li>
<pre><code class="language-bash">sudo pacman -S nmtui
</code></pre>
<p><code>nmtui</code>を実行すると、CUIベースのメニューが表示され、接続設定やアクティベートが行えます。</p>
<h2>4. トラブルシューティング</h2>
<h3>4.1 NetworkManagerが起動しない</h3>
<p>サービスの状態を確認します。</p>
<pre><code class="language-bash">sudo systemctl status NetworkManager
</code></pre>
<p>エラーがあれば、ジャーナルを確認。</p>
<pre><code class="language-bash">journalctl -u NetworkManager -n 50 --no-pager
</code></pre>
<p>よくある原因は、他のネットワークサービスとの競合です。<code>systemd-networkd</code>や<code>netctl</code>が有効化されていないか確認してください。</p>
<h3>4.2 Wi-Fiが認識されない</h3>
<p>無線デバイスがブロックされていないか確認。</p>
<pre><code class="language-bash">rfkill list
</code></pre>
<p>ソフトブロックされている場合は、<code>rfkill unblock wifi</code>で解除。</p>
<p>また、無線ファームウェアが不足している可能性があります。<code>dmesg | grep firmware</code>でエラーがないか確認し、必要なら<code>linux-firmware</code>をインストール。</p>
<h3>4.3 DHCPでIPアドレスが取得できない</h3>
<p>有線接続でDHCPがうまくいかない場合、手動でIPを設定してみてください。</p>
<pre><code class="language-bash">nmcli connection modify "プロファイル名" ipv4.method manual ipv4.addresses 192.168.1.100/24 ipv4.gateway 192.168.1.1 ipv4.dns 8.8.8.8
nmcli connection up "プロファイル名"
</code></pre>
<p>ルーターのDHCPサーバーが正しく動作しているかも確認しましょう。</p>
<h3>4.4 DNSが解決しない</h3>
<p><code>/etc/resolv.conf</code>がNetworkManagerによって管理されているか確認。</p>
<pre><code class="language-bash">ls -l /etc/resolv.conf
</code></pre>
<p>シンボリックリンクが<code>/run/NetworkManager/resolv.conf</code>などを指していればOK。もし手動で編集した場合は、<code>/etc/NetworkManager/NetworkManager.conf</code>に<code>dns=default</code>（または<code>dns=systemd-resolved</code>）と設定して再起動。</p>
