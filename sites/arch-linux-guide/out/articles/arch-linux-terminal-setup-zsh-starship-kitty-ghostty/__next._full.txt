11:T1f37,<h2>目標</h2>
<p>このチュートリアルでは、Arch Linux上で以下のターミナル環境を構築します。</p>
<li><strong>シェル</strong>: zsh（Z Shell）</li>
<li><strong>プロンプト</strong>: Starship（高速でカスタマイズ可能なプロンプト）</li>
<li><strong>ターミナルエミュレータ</strong>: Kitty または Ghostty（GPUアクセラレーション対応、高速）</li>
<p>最終的に、美しく情報量が多く、かつ高速なターミナル環境を手に入れます。</p>
<h2>前提条件</h2>
<li>Arch Linuxがインストールされ、インターネットに接続されていること</li>
<li>パッケージマネージャ（pacman）とAURヘルパー（yay または paru）が使えること</li>
<li>基本的なターミナル操作に慣れていること</li>
<h2>Step-by-step</h2>
<h3>1. 必要なパッケージのインストール</h3>
<p>まず、zsh、Starship、Kitty、Ghosttyをインストールします。KittyとGhosttyはどちらか一方でも両方でも構いません。</p>
<pre><code class="language-bash">sudo pacman -S zsh starship kitty ghostty
</code></pre>
<p>GhosttyはAURからもインストールできます（最新版が必要な場合）。</p>
<pre><code class="language-bash">yay -S ghostty
</code></pre>
<h3>2. デフォルトシェルをzshに変更</h3>
<p>現在のシェルを確認し、zshに変更します。</p>
<pre><code class="language-bash">echo $SHELL          # 現在のシェルを表示
chsh -s /usr/bin/zsh # デフォルトシェルをzshに変更
</code></pre>
<p>変更後、一度ログアウトして再ログインするか、新しいターミナルを開いて反映させます。</p>
<h3>3. zshの設定（.zshrc）</h3>
<p>ホームディレクトリに <code>.zshrc</code> ファイルを作成し、基本的な設定を追加します。</p>
<pre><code class="language-bash">nano ~/.zshrc
</code></pre>
<p>以下を記述します。</p>
<pre><code class="language-zsh">&lt;h1&gt;Starshipの初期化&lt;/h1&gt;
eval "$(starship init zsh)"
<p>&lt;h1&gt;プラグインマネージャ（zinitなど）を使う場合はここに追加&lt;/h1&gt;
&lt;h1&gt;例: zinit light zsh-users/zsh-autosuggestions&lt;/h1&gt;</p>
<p>&lt;h1&gt;エイリアス&lt;/h1&gt;
alias ll='ls -lah'
alias la='ls -A'
alias l='ls -CF'</p>
<p>&lt;h1&gt;履歴設定&lt;/h1&gt;
export HISTFILE=~/.zsh_history
export HISTSIZE=100000
export SAVEHIST=100000
setopt SHARE_HISTORY</p>
<p>&lt;h1&gt;補完機能&lt;/h1&gt;
zstyle ':completion:*' menu select
autoload -Uz compinit
compinit
</code></pre></p>
<p>保存後、設定を読み込みます。</p>
<pre><code class="language-bash">source ~/.zshrc
</code></pre>
<h3>4. Starshipの設定</h3>
<p>Starshipの設定ファイルを作成します。</p>
<pre><code class="language-bash">mkdir -p ~/.config &amp;&amp; touch ~/.config/starship.toml
</code></pre>
<p><code>~/.config/starship.toml</code> を編集して好みのプロンプトにカスタマイズします。以下はシンプルな例です。</p>
<pre><code class="language-toml">&lt;h1&gt;プロンプトのフォーマット&lt;/h1&gt;
format = """
[](#9A348E)
$os$
$username$
[](#DA627D)
$directory$
[](#FCA17D)
$git_branch$
$git_status$
$c$
$rust$
$golang$
$node$
$elixir$
$python$
[](#86BBD8)
$time$
[](#06969A)
$line_break$
$character"""
<p>&lt;h1&gt;シンボルを変更&lt;/h1&gt;
[character]
success_symbol = "[" # 通常のプロンプト記号
error_symbol = "[❌]" # エラー時</p>
<p>&lt;h1&gt;ディレクトリの表示&lt;/h1&gt;
[directory]
truncation_length = 3
truncate_to_repo = true</p>
<p>&lt;h1&gt;Git情報&lt;/h1&gt;
[git_branch]
symbol = "🌱 "
style = "bg:#9A348E"
format = '<a href="$style" target="_blank" rel="noopener">$symbol$branch</a>'</p>
<p>[git_status]
conflicted = "🏳"
ahead = "🏎💨"
behind = "😰"
diverged = "🔱"
untracked = "🤷"
stashed = "📦"
modified = "📝"
staged = '<a href="green" target="_blank" rel="noopener">++($count)</a>'
renamed = "👅"
deleted = "🗑"
</code></pre></p>
<p>設定後、新しいターミナルを開くか <code>source ~/.zshrc</code> で反映します。</p>
<h3>5. Kittyの設定</h3>
<p>Kittyの設定ファイルを作成します。</p>
<pre><code class="language-bash">mkdir -p ~/.config/kitty &amp;&amp; nano ~/.config/kitty/kitty.conf
</code></pre>
<p>以下は設定例です。</p>
<pre><code class="language-">&lt;h1&gt;フォント設定&lt;/h1&gt;
font_family      Fira Code
bold_font         Fira Code Bold
italic_font       Fira Code Italic
bold_italic_font Fira Code Bold Italic
font_size 12.0
<p>&lt;h1&gt;カラースキーム（Tokyo Night）&lt;/h1&gt;
include /path/to/tokyo-night.conf</p>
<p>&lt;h1&gt;キーバインド&lt;/h1&gt;
map ctrl+shift+c copy_to_clipboard
map ctrl+shift+v paste_from_clipboard
map ctrl+shift+up scroll_line_up
map ctrl+shift+down scroll_line_down</p>
<p>&lt;h1&gt;透明度&lt;/h1&gt;
background_opacity 0.9</p>
<p>&lt;h1&gt;その他&lt;/h1&gt;
enable_audio_bell no
</code></pre></p>
<p>カラースキームは自分で用意するか、<a href="https://github.com/dexpota/kitty-themes" target="_blank" rel="noopener">kitty-themes</a>からダウンロードして <code>include</code> で指定します。</p>
<h3>6. Ghosttyの設定</h3>
<p>Ghosttyの設定ファイルは <code>~/.config/ghostty/config</code> です。</p>
<pre><code class="language-bash">mkdir -p ~/.config/ghostty &amp;&amp; nano ~/.config/ghostty/config
</code></pre>
<p>設定例：</p>
<pre><code class="language-">&lt;h1&gt;フォント&lt;/h1&gt;
font-family = Fira Code
font-size = 12
<p>&lt;h1&gt;カラースキーム（Tokyo Night）&lt;/h1&gt;
theme = tokyo-night</p>
<p>&lt;h1&gt;透明度&lt;/h1&gt;
background-opacity = 0.9</p>
<p>&lt;h1&gt;キーバインド&lt;/h1&gt;
keybind = ctrl+shift+c:copy_to_clipboard
keybind = ctrl+shift+v:paste_from_clipboard
keybind = ctrl+shift+up:scroll_line_up
keybind = ctrl+shift+down:scroll_line_down</p>
<p>&lt;h1&gt;その他&lt;/h1&gt;
bell = false
</code></pre></p>
<p>Ghosttyは起動時に自動で設定を読み込みます。</p>
<h2>動作確認</h2>
<li>ターミナルエミュレータ（KittyまたはGhostty）を起動します。</li>
<li>プロンプトがStarshipの設定通りに表示されることを確認します。</li>
<li><code>echo $SHELL</code> で <code>/usr/bin/zsh</code> が表示されることを確認します。</li>
<li><code>ls</code> や <code>cd</code> などのコマンドが正常に動作するか確認します。</li>
<li>Gitリポジトリ内でブランチ情報が表示されるか確認します。</li>
