f:T1efd,<h2>はじめに</h2>
<p>この記事では、Arch Linuxのインストール時にLUKS（Linux Unified Key Setup）とdm-cryptを使ってディスク全体を暗号化する方法を解説します。対象読者はArch Linuxを初めてインストールする方で、ディスク暗号化の基礎知識があると理解がスムーズです。この手順により、ノートPCを紛失したり盗難に遭った場合でも、データの機密性を守ることができます。</p>
<h2>前提条件</h2>
<li>Arch Linuxのインストールメディア（USBなど）で起動していること</li>
<li>ターゲットディスク（例：/dev/sda）が空であること（または既存データをバックアップ済み）</li>
<li>インターネット接続が利用可能であること</li>
<h2>手順</h2>
<h3>ステップ1: パーティションの作成</h3>
<p>まず、暗号化するパーティションを作成します。ここではUEFIブート用にEFIシステムパーティション（ESP）と、暗号化するルートパーティションを作成します。</p>
<pre><code class="language-bash">&lt;h1&gt;ディスクをGPTでパーティショニング&lt;/h1&gt;
gdisk /dev/sda
</code></pre>
<p><code>gdisk</code>内で以下の操作を行います：
<li><code>o</code> で新しいGPTテーブルを作成</li>
<li><code>n</code> でパーティション1：サイズ512MiB、タイプコード <code>ef00</code>（EFI System）</li>
<li><code>n</code> でパーティション2：残り全て、タイプコード <code>8300</code>（Linux filesystem）</li>
<li><code>w</code> で書き込み</li></p>
<h3>ステップ2: LUKSコンテナの作成</h3>
<p>次に、ルートパーティションにLUKS暗号化を設定します。</p>
<pre><code class="language-bash">&lt;h1&gt;LUKSコンテナを作成（パスフレーズを設定）&lt;/h1&gt;
cryptsetup luksFormat /dev/sda2
&lt;h1&gt;コンテナを開く（/dev/mapper/cryptrootとしてマッピング）&lt;/h1&gt;
cryptsetup open /dev/sda2 cryptroot
</code></pre>
<p><code>luksFormat</code>実行時にパスフレーズを求められます。強力なパスフレーズを設定してください。</p>
<h3>ステップ3: ファイルシステムの作成とマウント</h3>
<p>暗号化されたデバイスにファイルシステムを作成し、マウントします。</p>
<pre><code class="language-bash">&lt;h1&gt;ファイルシステムの作成（ext4推奨）&lt;/h1&gt;
mkfs.ext4 /dev/mapper/cryptroot
&lt;h1&gt;ESPのファイルシステム作成（FAT32）&lt;/h1&gt;
mkfs.fat -F32 /dev/sda1
&lt;h1&gt;マウント&lt;/h1&gt;
mount /dev/mapper/cryptroot /mnt
mkdir /mnt/boot
mount /dev/sda1 /mnt/boot
</code></pre>
<h3>ステップ4: Arch Linuxの基本インストール</h3>
<p><code>pacstrap</code>でベースシステムをインストールします。</p>
<pre><code class="language-bash">&lt;h1&gt;必須パッケージのインストール&lt;/h1&gt;
pacstrap /mnt base linux linux-firmware vim
&lt;h1&gt;fstabの生成&lt;/h1&gt;
genfstab -U /mnt &gt;&gt; /mnt/etc/fstab
&lt;h1&gt;ルートディレクトリの変更&lt;/h1&gt;
arch-chroot /mnt
</code></pre>
<h3>ステップ5: システム設定とmkinitcpio</h3>
<p>chroot内で、mkinitcpioにdm-cryptとLUKSのフックを追加します。</p>
<pre><code class="language-bash">&lt;h1&gt;/etc/mkinitcpio.confを編集&lt;/h1&gt;
vim /etc/mkinitcpio.conf
</code></pre>
<p><code>HOOKS</code>行を以下のように変更します：</p>
<pre><code class="language-">HOOKS=(base udev autodetect modconf block keyboard encrypt filesystems fsck)
</code></pre>
<p><code>encrypt</code>フックを<code>filesystems</code>の前に追加してください。</p>
<pre><code class="language-bash">&lt;h1&gt;initramfsを再生成&lt;/h1&gt;
mkinitcpio -p linux
</code></pre>
<h3>ステップ6: ブートローダーの設定</h3>
<p>GRUBを使用する場合、<code>/etc/default/grub</code>にカーネルパラメータを追加します。</p>
<pre><code class="language-bash">&lt;h1&gt;パーティションのUUIDを確認&lt;/h1&gt;
blkid /dev/sda2
&lt;h1&gt;/etc/default/grubを編集&lt;/h1&gt;
vim /etc/default/grub
</code></pre>
<p><code>GRUB_CMDLINE_LINUX</code>行に以下を追加（UUIDは実際の値に置き換え）：</p>
<pre><code class="language-">GRUB_CMDLINE_LINUX="cryptdevice=UUID=xxxx-xxxx-xxxx-xxxx-xxxx:cryptroot root=/dev/mapper/cryptroot"
</code></pre>
<pre><code class="language-bash">&lt;h1&gt;GRUBをインストール&lt;/h1&gt;
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg
</code></pre>
<h3>ステップ7: 再起動</h3>
<p>chrootを抜けて再起動します。</p>
<pre><code class="language-bash">exit
umount -R /mnt
reboot
</code></pre>
<p>再起動後、LUKSのパスフレーズを求められます。正しく入力するとシステムが起動します。</p>
<h2>トラブルシューティング</h2>
<h3>実体験：日本語キーボードでパスフレーズが通らない</h3>
<p>筆者自身が経験した問題です。LUKSのパスフレーズを設定した時は日本語キーボード（jp106）を使っていましたが、initramfsの段階ではキーマップがUS配列になっているため、記号を含むパスフレーズが正しく入力できずロックアウトされました。</p>
<p><strong>対策</strong>:
<li><code>/etc/mkinitcpio.conf</code> の <code>HOOKS</code> に <code>keymap</code> を追加する</li>
<li><code>mkinitcpio -P</code> でinitramfsを再生成する</li>
<li>パスフレーズはASCII文字のみで構成する（記号を使わない）</li></p>
<p>この設定で再起動後、日本語キーボードで正しくパスフレーズが通るようになりました。</p>
<h3>エラー: カーネルパニック「not syncing: VFS: Unable to mount root fs」</h3>
<p><strong>原因</strong>: initramfsにencryptフックが含まれていない、またはカーネルパラメータが間違っている。</p>
<p><strong>対処</strong>: chroot環境で<code>/etc/mkinitcpio.conf</code>を確認し、<code>mkinitcpio -p linux</code>を再実行。また、<code>/etc/default/grub</code>のUUIDが正しいか確認。</p>
<h3>エラー: 「cryptsetup: ERROR: cryptroot: cryptsetup failed, bad password or options?」</h3>
<p><strong>原因</strong>: パスフレーズが間違っている、またはLUKSコンテナが正しく開かれていない。</p>
<p><strong>対処</strong>: 起動時にパスフレーズを注意深く入力。キーボードレイアウトが正しいか確認（例：日本語キーボードなら<code>loadkeys jp106</code>）。</p>
<h3>エラー: GRUBインストール時に「efibootmgr: not found」</h3>
<p><strong>原因</strong>: <code>efibootmgr</code>がインストールされていない。</p>
<p><strong>対処</strong>: <code>pacman -S efibootmgr</code>でインストールしてから再度<code>grub-install</code>を実行。</p>
